# دليل النشر

## نظام مركز البيانات المركزي عبر الولايات القضائية وسجل الائتمان الإصدار 2.5

**مُعد لصالح:** Systems In Motion Limited  
**إصدار المستند:** 2.5  
**التاريخ:** أبريل 2026

---

## 1. نظرة عامة

يقدم هذا الدليل تعليمات خطوة بخطوة لنشر نظام سجل الائتمان. يتكون التطبيق من واجهة أمامية React يقدمها خادم Express.js خلفي، مع PostgreSQL كقاعدة بيانات. يغطي هذا الدليل سيناريوهين للنشر: النشر على Replit والنشر العام على Linux/Docker.

## 4. إعداد قاعدة البيانات

### 4.1 توفير PostgreSQL

يتطلب النظام قاعدة بيانات PostgreSQL. تشمل الخيارات:

- **Neon** (PostgreSQL بدون خادم، موصى به لـ Replit)
- **Amazon RDS** لأعباء العمل الإنتاجية
- **PostgreSQL مستضاف ذاتياً** على خوادم Linux
- **Docker PostgreSQL** للنشر في حاويات

### 4.2 إضافات PostgreSQL

يتطلب التطبيق إضافة PostgreSQL التالية:

- **pg_trgm** — التشابه ثلاثي الأحرف للمطابقة الضبابية للكيانات (ENT-02). يتم تفعيلها تلقائياً عند بدء التشغيل عبر `CREATE EXTENSION IF NOT EXISTS pg_trgm`.

تأكد من أن مستخدم PostgreSQL لديه صلاحية إنشاء الإضافات، أو قم بإنشائها مسبقاً:
```sql
CREATE EXTENSION IF NOT EXISTS pg_trgm;
```

### 4.3 تهيئة المخطط

يستخدم التطبيق Drizzle ORM لإدارة المخطط. عند التشغيل الأول، سيقوم التطبيق تلقائياً بـ:

1. الاتصال بقاعدة البيانات باستخدام `DATABASE_URL`
2. تفعيل إضافة `pg_trgm` للمطابقة الضبابية
3. دفع المخطط باستخدام Drizzle (21 جدولاً)
4. تغذية البيانات الأولية (المستخدمون، المقترضون، حسابات الائتمان، إلخ.)

لدفع المخطط يدوياً:

```bash
npx drizzle-kit push
```

## 5. عملية البناء

### 5.1 تثبيت التبعيات

```bash
npm install
```

### 5.2 البناء للإنتاج

```bash
npm run build
```

ينفذ هذا خطوتي بناء:

1. **بناء الخلفية** (esbuild): يُجمّع كود TypeScript للخادم إلى `dist/index.cjs`
2. **بناء الواجهة الأمامية** (Vite): يُجمّع تطبيق React إلى `dist/public/`

### 5.3 هيكل مخرجات البناء

```
dist/
  index.cjs          # حزمة الخادم المُجمّعة (CommonJS)
  public/             # أصول الواجهة الأمامية المُجمّعة
    index.html
    assets/
      *.js            # حزم JavaScript
      *.css           # حزم الأنماط
uploads/              # الملفات المرفوعة من المستخدمين (تُنشأ أثناء التشغيل)
  photos/             # صور المقترضين الشخصية (5MB كحد أقصى لكل صورة)
  documents/          # وثائق هوية المقترضين (10MB كحد أقصى لكل وثيقة)
```

> **ملاحظة:** يتم إنشاء مجلد `uploads/` تلقائياً عند بدء تشغيل التطبيق. تأكد من أن عملية التطبيق لديها صلاحيات الكتابة في مجلد العمل. لنشر Docker، فكر في تركيب وحدة تخزين دائمة في مسار `uploads/` للحفاظ على الملفات المرفوعة عبر إعادة تشغيل الحاوية.

## 6. التشغيل في الإنتاج

### 6.1 بدء تشغيل التطبيق

```bash
NODE_ENV=production node dist/index.cjs
```

سيقوم الخادم بـ:
1. الاتصال بـ PostgreSQL
2. تشغيل عملية التغذية (متساوية القوة، تتخطى إذا وُجدت البيانات)
3. تقديم الواجهة الأمامية من `dist/public/`
4. الاستماع على المنفذ المُكوّن (الافتراضي 5000)

### 6.2 إدارة العمليات

للإنتاج، استخدم مدير عمليات:

**باستخدام PM2:**
```bash
npm install -g pm2
NODE_ENV=production pm2 start dist/index.cjs --name credit-registry
pm2 save
pm2 startup
```

**باستخدام systemd:**
```ini
[Unit]
Description=Credit Registry System
After=network.target

[Service]
Type=simple
User=appuser
WorkingDirectory=/opt/credit-registry
Environment=NODE_ENV=production
Environment=DATABASE_URL=postgresql://...
Environment=SESSION_SECRET=your-secret-here
ExecStart=/usr/bin/node dist/index.cjs
Restart=on-failure
RestartSec=10

[Install]
WantedBy=multi-user.target
```

## 7. النشر الخاص بـ Replit

### 7.1 التكوين

التطبيق مُكوّن مسبقاً للنشر على Replit. يحدد ملف `.replit`:

- أمر البناء: `npm run build`
- أمر التشغيل: `node ./dist/index.cjs`
- هدف النشر: Autoscale

### 7.2 الأسرار البيئية

في Replit، قم بتعيين متغيرات البيئة عبر علامة التبويب Secrets:
1. `DATABASE_URL` - سلسلة اتصال PostgreSQL (يُوصى بـ Neon)
2. `SESSION_SECRET` - سلسلة عشوائية لتشفير الجلسة

### 7.3 وضع التطوير

في وضع التطوير، يعمل التطبيق مع Vite HMR (الاستبدال السريع للوحدات):

```bash
npm run dev
```

يبدأ هذا:
- خادم Express API مع TypeScript عبر tsx
- خادم تطوير Vite مع وكيل HMR

## 8. النشر باستخدام Docker

### 8.1 Dockerfile

```dockerfile
FROM node:20-alpine AS builder

WORKDIR /app
COPY package*.json ./
RUN npm ci
COPY . .
RUN npm run build

FROM node:20-alpine AS runner

WORKDIR /app
COPY --from=builder /app/dist ./dist
COPY --from=builder /app/node_modules ./node_modules
COPY --from=builder /app/package.json ./

ENV NODE_ENV=production
ENV PORT=5000

EXPOSE 5000

CMD ["node", "dist/index.cjs"]
```

### 8.2 Docker Compose

```yaml
version: '3.8'
services:
  app:
    build: .
    ports:
      - "5000:5000"
    environment:
      - NODE_ENV=production
      - DATABASE_URL=postgresql://postgres:password@db:5432/credit_registry
      - SESSION_SECRET=your-production-secret-here
    volumes:
      - uploads:/app/uploads
    depends_on:
      - db
    restart: unless-stopped

  db:
    image: postgres:16-alpine
    environment:
      - POSTGRES_DB=credit_registry
      - POSTGRES_USER=postgres
      - POSTGRES_PASSWORD=password
    volumes:
      - pgdata:/var/lib/postgresql/data
    ports:
      - "5432:5432"

volumes:
  pgdata:
  uploads:
```

### 8.3 البناء والتشغيل

```bash
docker-compose up -d --build
```

## 10. تكوين الوكيل العكسي

### 10.1 Nginx

```nginx
server {
    listen 80;
    server_name credit-registry.example.com;

    location / {
        proxy_pass http://127.0.0.1:5000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
```

## 12. النسخ الاحتياطي والاسترداد

### 12.1 النسخ الاحتياطي لقاعدة البيانات

```bash
pg_dump $DATABASE_URL > backup_$(date +%Y%m%d_%H%M%S).sql
```

### 12.2 النسخ الاحتياطي التلقائي

```bash
# إدخال crontab للنسخ الاحتياطي اليومي في الساعة 2 صباحاً

```

### 12.3 الاسترداد من نسخة احتياطية

```bash
psql $DATABASE_URL < backup_20250115_020000.sql
```

## 13. استكشاف الأخطاء وإصلاحها

### 13.1 المشكلات الشائعة

| المشكلة | السبب | الحل |
|---------|-------|------|
| رفض الاتصال على المنفذ 5000 | التطبيق لا يعمل | تحقق من `NODE_ENV`، تأكد من وجود `dist/index.cjs` |
| خطأ في اتصال قاعدة البيانات | `DATABASE_URL` غير صالح | تحقق من سلسلة الاتصال، افحص قواعد الشبكة/جدار الحماية |
| الجلسة لا تستمر | `SESSION_SECRET` مفقود | عيّن متغير البيئة `SESSION_SECRET` |
| 440 انتهاء الجلسة | مهلة الخمول 15 دقيقة | أعد تسجيل الدخول؛ هذا سلوك أمني متوقع (NFR-SEC-09) |
| فشل التغذية عند التشغيل الأول | الجداول لم تُنشأ بعد | شغّل `npx drizzle-kit push` أولاً، ثم أعد التشغيل |
| فشل البناء | تبعيات مفقودة | شغّل `npm install` قبل `npm run build` |
| الملفات الثابتة لا تُقدّم | `NODE_ENV` خاطئ | تأكد من `NODE_ENV=production` لتقديم الملفات الثابتة |
| API يُعيد 401 | غير مُصادق | سجّل الدخول أولاً؛ تحقق من إرسال ملف تعريف ارتباط الجلسة |
| API يُعيد 403 | صلاحيات غير كافية | تحقق من تطابق دور المستخدم مع متطلبات المسار |

### 13.2 التصحيح

### 13.3 ضبط الأداء

- زيادة حجم مجمع قاعدة البيانات لزيادة التزامن
- تفعيل تجميع اتصالات PostgreSQL (PgBouncer)
- استخدام CDN لأصول الواجهة الأمامية الثابتة
- ضغط gzip مُدمج في التطبيق عبر وسيط `compression` (ENT-05)؛ ضغط الوكيل العكسي اختياري
- مراقبة وفهرسة الأعمدة المُستعلم عنها بكثرة
- تقسيم الكود عبر React.lazy يقلل حجم الحزمة الأولية لتحميل أسرع للصفحات (ENT-05)

## 15. تبعيات التحسينات المؤسسية

الحزم الإضافية التالية تمت إضافتها للتحسينات المؤسسية (الإصدار 1.1 و 1.2):

|--------|--------|---------|
