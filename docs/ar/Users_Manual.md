## 1. المقدمة

### 1.1 الغرض من النظام

نظام السجل الائتماني هو تطبيق ويب طوّرته شركة Systems In Motion Limited لإدارة المعلومات الائتمانية وسجلات المقترضين وتقييم المخاطر الائتمانية عبر البنوك التجارية ومؤسسات التمويل الأصغر ومقدمي الخدمات المالية الأخرى. يعمل كمركز بيانات مركزي عابر للولايات القضائية (CDH) يُمكّن من الرقابة التنظيمية وإدارة المخاطر الائتمانية وحماية المستهلك.

### 1.2 الولايات القضائية المدعومة

يدعم النظام النشر عبر جميع **54 دولة أفريقية**، مجمّعة حسب المنطقة:

**غرب أفريقيا (ECOWAS):**
بنين، بوركينا فاسو، الرأس الأخضر، كوت ديفوار، غامبيا، غانا، غينيا، غينيا بيساو، ليبيريا، مالي، النيجر، نيجيريا، السنغال، سيراليون، توغو

**شرق أفريقيا (EAC):**
بوروندي، جمهورية الكونغو الديمقراطية، إثيوبيا، كينيا، رواندا، جنوب السودان، تنزانيا، أوغندا

**جنوب أفريقيا (SADC):**
أنغولا، بوتسوانا، إسواتيني، ليسوتو، مدغشقر، ملاوي، موريشيوس، موزمبيق، ناميبيا، سيشل، جنوب أفريقيا، زامبيا، زيمبابوي

**وسط أفريقيا (CEMAC):**
الكاميرون، جمهورية أفريقيا الوسطى، تشاد، الكونغو، غينيا الاستوائية، الغابون، ساو تومي وبرينسيبي

**شمال أفريقيا (AMU):**
الجزائر، جيبوتي، مصر، إريتريا، ليبيا، موريتانيا، المغرب، الصومال، السودان، تونس، جزر القمر

### 1.3 اللغات المدعومة

يدعم النظام خمس لغات:

- **الإنجليزية (EN)** — اللغة الافتراضية
- **الفرنسية (FR)** — ترجمة فرنسية كاملة متوفرة
- **البرتغالية (PT)** — ترجمة برتغالية كاملة متوفرة
- **العربية (AR)** — ترجمة عربية كاملة متوفرة
- **السواحلية (SW)** — ترجمة سواحلية كاملة متوفرة

يمكن للمستخدمين التبديل بين اللغات في أي وقت باستخدام محوّل اللغة المتاح في صفحة تسجيل الدخول وفي شريط التطبيق العلوي.

## 2. البدء

### 2.1 الوصول إلى النظام

افتح متصفح ويب حديث (Chrome أو Firefox أو Safari أو Edge) وانتقل إلى عنوان URL الخاص بالنظام الذي يوفره المسؤول. ستظهر صفحة تسجيل الدخول.

### 2.2 تسجيل الدخول

1. في صفحة تسجيل الدخول، اختر **اللغة المفضلة** (EN/FR/PT) باستخدام محوّل اللغة إذا رغبت.
2. أدخل **اسم المستخدم** في حقل اسم المستخدم.
3. أدخل **كلمة المرور** في حقل كلمة المرور.
4. انقر على زر **تسجيل الدخول**.
5. عند المصادقة الناجحة، ستتم إعادة توجيهك إلى لوحة المعلومات.

**ملاحظات مهمة:**
- يتم قفل الحسابات لمدة **15 دقيقة** بعد **3 محاولات فاشلة متتالية** لتسجيل الدخول.
- إذا تم قفل حسابك، انتظر حتى انتهاء فترة القفل قبل المحاولة مرة أخرى.
- إذا كان حسابك معلقاً أو غير مفعّل، تواصل مع مسؤول النظام.

### 2.3 تغيير كلمة المرور لأول مرة

إذا قام المسؤول بتحديد حسابك لتغيير إلزامي لكلمة المرور (أو إذا كان عمر كلمة المرور أكثر من 90 يوماً)، ستتم مطالبتك بتغيير كلمة المرور عند تسجيل الدخول.

**متطلبات كلمة المرور:**
- 8 أحرف كحد أدنى
- حرف كبير واحد على الأقل (A-Z)
- حرف صغير واحد على الأقل (a-z)
- رقم واحد على الأقل (0-9)
- حرف خاص واحد على الأقل (!@#$%^&* إلخ)

لتغيير كلمة المرور:
1. سيظهر مربع حوار تغيير كلمة المرور تلقائياً إذا لزم الأمر.
2. أدخل **كلمة المرور الحالية**.
3. أدخل **كلمة المرور الجديدة** مع استيفاء المتطلبات أعلاه.
4. انقر على **تغيير كلمة المرور**.

يمكنك أيضاً تغيير كلمة المرور في أي وقت بالنقر على خيار تغيير كلمة المرور في الشريط الجانبي.

### 2.4 تبديل اللغة (EN/FR/PT)

لتبديل لغة الواجهة:
1. حدد موقع زر محوّل اللغة في شريط العنوان العلوي أو في صفحة تسجيل الدخول (يُظهر "EN" أو "FR" أو "PT").
2. انقر على الزر للتبديل بين الإنجليزية والفرنسية والبرتغالية.
3. ستتحدث الواجهة بالكامل فوراً إلى اللغة المختارة.

### 2.5 تبديل السمة (فاتح/داكن)

يدعم النظام السمتين الفاتحة والداكنة:
1. انقر على أيقونة تبديل السمة (شمس/قمر) في شريط العنوان العلوي.
2. ستتبدل الواجهة بين الوضع الفاتح والداكن.
3. يتم حفظ تفضيلك في التخزين المحلي للمتصفح.

### 2.6 فهم التنقل في الشريط الجانبي

يوفر الشريط الجانبي الوصول إلى جميع وحدات النظام، منظمة في ثلاثة أقسام:

**الرئيسي:**
- لوحة المعلومات
- المقترضون
- الحسابات الائتمانية
- البحث الائتماني
- التحميل الجماعي

**التقارير والامتثال:**
- الموافقات المعلقة
- النزاعات
- سجل التدقيق
- إدارة الموافقات
- الفوترة
- مكتب المساعدة

**الإدارة:**
- إدارة المستخدمين (المسؤول فقط)
- المؤسسات (المسؤول فقط)
- مفاتيح API (المسؤول فقط)
- أسعار الصرف (المسؤول فقط)
- إدارة API (المسؤول فقط)
- سياسات الاحتفاظ (المسؤول/المنظم)

يمكن طي الشريط الجانبي بالنقر على زر تبديل الشريط الجانبي في العنوان. في وضع الطي، تظهر الأيقونات فقط. على الأجهزة المحمولة، يُفتح الشريط الجانبي كلوحة منزلقة.

### 2.7 المصادقة متعددة العوامل (MFA)

يدعم النظام المصادقة متعددة العوامل الاختيارية المبنية على TOTP لتعزيز الأمان. عند تمكين MFA:
1. بعد إدخال اسم المستخدم وكلمة المرور، ستتم مطالبتك برمز من 6 أرقام.
2. افتح تطبيق المصادقة (Google Authenticator أو Authy أو غيره) للحصول على الرمز.
3. أدخل الرمز وانقر على **تحقق** لإكمال تسجيل الدخول.

راجع القسم 21 للحصول على تعليمات إعداد MFA المفصلة.

### 2.8 انتهاء مهلة الجلسة (تسجيل خروج تلقائي بعد 15 دقيقة)

للامتثال الأمني (NFR-SEC-09)، يقوم النظام تلقائياً بتسجيل خروجك بعد **15 دقيقة من عدم النشاط**. عند انتهاء مهلة الجلسة:
- ستتم إعادة توجيهك إلى صفحة تسجيل الدخول.
- سيتم فقدان أي عمل غير محفوظ.
- تنطبق مدة جلسة قصوى تبلغ **8 ساعات** بغض النظر عن النشاط.

## 3. لوحة المعلومات

توفر لوحة المعلومات نظرة سريعة على المقاييس الرئيسية لنظام السجل الائتماني والنشاط الأخير.

### 3.1 فهم بطاقات الإحصائيات الثمانية

تعرض لوحة المعلومات 8 بطاقات إحصائيات تفاعلية:

| البطاقة | الوصف |
|---------|-------|
| **إجمالي المقترضين** | العدد الإجمالي لسجلات المقترضين المسجلين (أفراد وشركات) |
| **الحسابات الائتمانية** | العدد الإجمالي لحسابات الائتمان/القروض في النظام |
| **المستحق** | إجمالي الرصيد المستحق عبر جميع الحسابات النشطة |
| **المتأخر** | عدد الحسابات ذات الدفعات المتأخرة |
| **المتعثر** | عدد الحسابات غير العاملة / المتعثرة |
| **الاستعلامات** | العدد الإجمالي للاستعلامات/عمليات البحث الائتمانية المنفذة |
| **الموافقات المعلقة** | عدد طلبات الموافقة المعلقة في نظام صانع-مدقق |
| **النزاعات المفتوحة** | عدد حالات النزاع المفتوحة حالياً |

### 3.2 النقر على البطاقات لعرض التفاصيل

كل بطاقة إحصائيات قابلة للنقر. يؤدي النقر على بطاقة إلى فتح لوحة تفاصيل تعرض السجلات الأساسية لذلك المقياس:

1. انقر على أي بطاقة إحصائيات (مثل "المتأخر").
2. ستظهر لوحة منزلقة تعرض قائمة بالسجلات ذات الصلة.
3. انقر على عناصر فردية في القائمة للانتقال إلى صفحة السجل المفصل.
4. أغلق لوحة التفاصيل بالنقر على زر الإغلاق أو بالنقر خارج اللوحة.

### 3.3 قسم النشاط الأخير

أسفل بطاقات الإحصائيات، تعرض لوحة المعلومات النشاط الأخير في النظام بما في ذلك:
- سجلات المقترضين المنشأة حديثاً
- الحسابات الائتمانية الجديدة
- إجراءات الموافقة الأخيرة
- تقديم النزاعات وحلها

### 3.4 رسم نمو المحفظة

أسفل قسم النشاط الأخير، تعرض لوحة المعلومات **رسم نمو المحفظة** — رسم بياني مساحي لمدة 12 شهراً يُصور اتجاهات المقترضين والحسابات الائتمانية بمرور الوقت.

1. يعرض الرسم سلسلتين مساحيتين متراكبتين:
   - **المقترضون** — العدد الشهري للمقترضين المسجلين
   - **الحسابات** — العدد الشهري للحسابات الائتمانية
2. يُظهر المحور الأفقي آخر 12 شهراً (مثل "Apr 2025"، "May 2025"، إلخ).
3. مرر المؤشر فوق أي نقطة على الرسم لرؤية تلميح يعرض الشهر الدقيق وعدد المقترضين وعدد الحسابات.
4. الرسم مبني بمكتبة Recharts وهو متجاوب بالكامل — يتكيف مع عرض الشاشة المتاح.
5. الألوان والتنسيق يتكيفان تلقائياً مع السمة الحالية (الوضع الفاتح أو الداكن).

### 3.5 رسوم حالة الحسابات وأنواع القروض

يظهر رسمان إضافيان بجانب رسم نمو المحفظة:

**رسم دائري مفرغ لحالة الحسابات:**
- رسم دائري مفرغ (حلقي) يُظهر توزيع الحسابات الائتمانية حسب الحالة (Current، Delinquent، Default، Closed، Restructured، Written Off).
- كل جزء ملون ومُسمى باسم الحالة والعدد.
- مرر المؤشر فوق جزء لرؤية العدد الدقيق لتلك الحالة.

**رسم أفقي شريطي لأنواع القروض:**
- رسم أفقي شريطي يُظهر توزيع الحسابات الائتمانية حسب نوع القرض (Personal Loan، Mortgage، Vehicle Loan، Business Loan، Corporate Loan، Overdraft، Credit Card، Microfinance).
- الأشرطة مرتبة حسب العدد، مما يسهل تحديد أنواع القروض الأكثر شيوعاً.
- مرر المؤشر فوق شريط لرؤية العدد الدقيق لنوع القرض.

### 3.6 خريطة أفريقيا

تتضمن لوحة المعلومات **خريطة SVG تفاعلية لأفريقيا** توفر نظرة جغرافية عامة على تغطية سجل الائتمان عبر القارة.

1. تعرض الخريطة جميع **الدول الأفريقية الـ 54** كمناطق SVG فردية.
2. الدول مُظللة بـ**تلوين حراري** بناءً على مستويات نشاط المقترضين:
   - التظليل الأغمق يشير إلى نشاط أعلى للمقترضين/الحسابات.
   - التظليل الأفتح يشير إلى نشاط أقل.
   - تظهر الدول التي لا يوجد فيها نشاط بلون محايد/فاتح.
3. **مرر المؤشر** فوق أي دولة لرؤية تلميح يعرض اسم الدولة وعدد المقترضين وعدد الحسابات.
4. يُعرض **مفتاح** أسفل الخريطة يوضح مقياس ألوان مستوى النشاط.
5. الخريطة متجاوبة بالكامل وتدعم الوضع الداكن.

## 4. إدارة المقترضين

تتيح وحدة إدارة المقترضين تسجيل ملفات المقترضين والبحث عنها وإدارتها.

### 4.1 البحث عن المقترضين (بالاسم/الهوية)

1. انتقل إلى **المقترضون** من الشريط الجانبي.
2. استخدم شريط البحث في أعلى الصفحة.
3. اكتب اسم المقترض أو رقم الهوية الوطنية أو معلومات تعريفية أخرى.
4. ستتم تصفية النتائج تلقائياً أثناء الكتابة.
5. انقر على بطاقة المقترض لعرض تفاصيله الكاملة.

### 4.2 تسجيل مقترض فرد

1. انقر على زر **تسجيل مقترض** (أعلى اليمين).
2. اختر نوع المقترض: **فرد**.
3. املأ الحقول المطلوبة:
   - **الاسم الأول** (مطلوب)
   - **اسم العائلة** (مطلوب)
   - **رقم الهوية الوطنية** (مطلوب، يجب أن يكون فريداً)
   - **تاريخ الميلاد**
   - **الجنس** (ذكر/أنثى)
   - **الهاتف**
   - **البريد الإلكتروني**
   - **العنوان**، **المدينة**، **المنطقة**
   - **اسم جهة العمل**، **المهنة**
   - **القطاع**
4. اختيارياً حدد خانة **علامة PEP** إذا كان المقترض شخصاً مكشوفاً سياسياً.
   - إذا تم تحديد PEP، قدم **تفاصيل PEP** تصف طبيعة التعرض السياسي.
5. اختر **المستوى التعليمي** (بدون، ابتدائي، ثانوي، دبلوم، بكالوريوس، ماجستير، دكتوراه).
6. أدخل اسم **المؤسسة التعليمية**.
7. أدخل **سجل التوظيف** في منطقة النص.
8. انقر على **تسجيل مقترض** للإرسال.
9. سيتم إرسال التسجيل لـ**موافقة صانع-مدقق** (راجع القسم 7).

### 4.3 تسجيل مقترض شركة

1. انقر على زر **تسجيل مقترض**.
2. اختر نوع المقترض: **شركة**.
3. املأ الحقول المطلوبة:
   - **اسم الشركة** (مطلوب)
   - **رقم الهوية الوطنية** (مطلوب، يجب أن يكون فريداً)
   - **رقم السجل التجاري**
   - **الهاتف**، **البريد الإلكتروني**
   - **العنوان**، **المدينة**، **المنطقة**
   - **القطاع**
4. اختيارياً حدد علامة PEP وقدم التفاصيل.
5. انقر على **تسجيل مقترض** للإرسال للموافقة.

### 4.4 وسم PEP

تشير علامة الشخص المكشوف سياسياً (PEP) إلى أن المقترض يشغل منصباً عاماً بارزاً أو مرتبط ارتباطاً وثيقاً بشخص يشغل هذا المنصب. لوسم مقترض كـ PEP:

1. أثناء التسجيل أو عند تعديل مقترض، حدد خانة **علامة PEP**.
2. أدخل التفاصيل في حقل **تفاصيل PEP** لشرح طبيعة التعرض السياسي.
3. يعرض المقترضون الموسومون بـ PEP شارة PEP حمراء على بطاقة ملفهم الشخصي.

### 4.5 عرض تفاصيل المقترض

1. انقر على أي بطاقة مقترض في قائمة المقترضين.
2. تعرض صفحة تفاصيل المقترض:
   - **صورة الملف الشخصي** — يتم عرض صورة رمزية مُنشأة تلقائياً (عبر DiceBear) بشكل افتراضي؛ يمكن تحميل صورة مخصصة بتمرير المؤشر فوق الصورة الرمزية والنقر على أيقونة الكاميرا
   - المعلومات الشخصية/معلومات الشركة
   - جميع الحسابات الائتمانية المرتبطة
   - الوصول إلى التقرير الائتماني
   - الأحكام القضائية
   - سجلات الموافقة
   - الأطراف ذات الصلة
   - **قسم مستند الهوية** — يعرض مسح الهوية المحمّل (جواز السفر أو الهوية الوطنية) أو رسالة تحميل إذا لم يتم تحميل أي مستند بعد. انقر على زر "تحميل الهوية" في العنوان أو منطقة التحميل في بطاقة المعلومات الشخصية لإرفاق مستند.

### 4.6 ربط الأطراف ذات الصلة

يمكن ربط المقترضين بأطراف ذات صلة بأنواع العلاقات التالية: spouse، guarantor، director، shareholder، beneficial_owner، subsidiary، وparent_company:

1. في صفحة تفاصيل المقترض، اعرض قسم **الأطراف ذات الصلة**.
2. يتم عرض المقترضين ذوي الصلة مع نوع علاقتهم.
3. انقر على طرف ذي صلة للانتقال إلى ملفه الشخصي كمقترض.

### 4.7 فهم صانع-مدقق (التغييرات تحتاج موافقة)

جميع طلبات إنشاء وتعديل المقترضين تمر عبر سير عمل صانع-مدقق:

- عند تقديم مقترض جديد أو تعديل مقترض قائم، يتم إنشاء **طلب موافقة معلق**.
- يجب على مستخدم مصرّح مختلف (مسؤول أو منظم) مراجعة التغيير والموافقة عليه.
- ستتلقى إشعاراً عند الموافقة على طلبك أو رفضه.
- راجع القسم 7 للحصول على التفاصيل الكاملة حول سير عمل صانع-مدقق.

### 4.8 التصفح بالصفحات

قائمة المقترضين مقسمة إلى صفحات للتعامل مع مجموعات البيانات الكبيرة (أكثر من 102,000 سجل عبر 54 دولة):
- استخدم زري **السابق** و**التالي** للتنقل بين الصفحات.
- يتم عرض رقم الصفحة الحالي والعدد الإجمالي في الأسفل.
- حجم الصفحة الافتراضي هو 50 سجلاً لكل صفحة.

## 5. الحسابات الائتمانية

تدير وحدة الحسابات الائتمانية القروض والتسهيلات الائتمانية المرتبطة بالمقترضين.

### 5.1 إضافة حسابات ائتمانية

1. انتقل إلى **الحسابات الائتمانية** من الشريط الجانبي.
2. انقر على زر **إضافة حساب**.
3. املأ الحقول المطلوبة:
   - **المقترض** — اختر من القائمة المنسدلة
   - **المؤسسة المُقرضة** — اسم المؤسسة المُقرضة
   - **رقم الحساب** — معرّف حساب فريد
   - **نوع الحساب** — اختر من: Personal Loan، Mortgage، Vehicle Loan، Business Loan، Corporate Loan، Overdraft، Credit Card، Microfinance
   - **المبلغ الأصلي** — مبلغ القرض/التسهيل الأصلي
   - **الرصيد الحالي** — الرصيد المستحق الحالي
   - **العملة** — اختر من أكثر من 45 عملة مدعومة (42 أفريقية بالإضافة إلى USD وEUR وGBP)
   - **سعر الفائدة** — نسبة سعر الفائدة السنوي
   - **الحالة** — Current، Delinquent، Default، Closed، Restructured
   - **تاريخ الصرف** — متى تم صرف القرض
   - **تاريخ الاستحقاق** — متى يستحق القرض
   - **نوع الضمان** — نوع الضمان (مثل عقار، مركبة، نقد)
   - **قيمة الضمان** — قيمة الضمان
4. حقول إضافية:
   - **بدون فائدة** — حدد إذا كان هذا تسهيلاً متوافقاً مع الشريعة أو بدون فائدة
   - **فترة السماح (أشهر)** — عدد أشهر فترة السماح
   - **عدد مرات إعادة الهيكلة** — عدد مرات إعادة هيكلة الحساب
5. انقر على **إنشاء حساب** للإرسال لموافقة صانع-مدقق.

### 5.2 فهم حالات الحسابات

| الحالة | الوصف |
|--------|-------|
| **Current** | الحساب يعمل بشكل طبيعي؛ جميع الدفعات محدثة |
| **Delinquent** | الحساب لديه دفعات متأخرة لكنه لم يتعثر بعد |
| **Default** | تم تصنيف الحساب كغير عامل |
| **Closed** | تم سداد الحساب بالكامل أو إغلاقه |
| **Restructured** | تم تعديل/إعادة هيكلة شروط القرض |
| **Written Off** | تم شطب الحساب كخسارة |

### 5.3 دعم العملات المتعددة

يدعم النظام أكثر من 42 عملة أفريقية بالإضافة إلى USD وEUR وGBP (أكثر من 45 إجمالاً) عبر جميع الأسواق الأفريقية والدولية. عند إنشاء أو عرض الحسابات الائتمانية:
- اختر العملة المناسبة من القائمة المنسدلة.
- يتم عرض المبالغ بتنسيق مناسب للعملة المختارة.
- راجع الملحق ج للحصول على القائمة الكاملة للعملات المدعومة.

### 5.4 عرض سجل المدفوعات (شبكة 12 فترة)

لكل حساب ائتماني سجل مدفوعات مرتبط يعرض شبكة أداء لـ 12 فترة:

1. انتقل إلى صفحة تفاصيل المقترض.
2. انقر على حساب ائتماني لعرض تفاصيله.
3. تعرض شبكة سجل المدفوعات:
   - **الفترة** — الشهر/السنة
   - **المبلغ المستحق** — مبلغ الدفع المجدول
   - **المبلغ المدفوع** — الدفع الفعلي المستلم
   - **الحالة** — في الوقت المحدد، متأخر، فائت، أو جزئي
   - **أيام التأخير** — عدد أيام تأخر الدفع (إن وُجد)

## 6. البحث الائتماني والتقارير

### 6.1 البحث عن مقترض واحد

1. انتقل إلى **البحث الائتماني** من الشريط الجانبي.
2. أدخل **رقم الهوية الوطنية** أو **اسم** المقترض في حقل البحث.
3. اختر **الغرض** من الاستعلام:
   - ائتمان جديد
   - مراجعة
   - تحصيل
   - تنظيمي
   - مراقبة المحفظة
4. أكد أنه تم تقديم **الموافقة**.
5. انقر على **بحث** لإجراء فحص المرجع الائتماني.
6. تعرض النتائج ملخص ائتمان المقترض بما في ذلك الحسابات النشطة والتقييم الائتماني وسجل الاستعلامات.

### 6.2 البحث الائتماني الجماعي (أرقام هوية متعددة)

1. انتقل إلى **البحث الائتماني**.
2. اختر خيار **البحث الجماعي**.
3. أدخل أرقام هوية وطنية متعددة (واحد في كل سطر أو مفصولة بفواصل).
4. اختر الغرض من الاستعلام.
5. انقر على **إرسال** لإجراء فحوصات مرجع ائتماني جماعية.
6. يتم إرجاع النتائج لكل معرّف، مع إظهار حالة الوجود/عدم الوجود.

### 6.3 إنشاء التقارير الائتمانية

1. من صفحة تفاصيل المقترض أو نتائج البحث الائتماني، انقر على **إنشاء تقرير ائتماني**.
2. اختر **الغرض** من التقرير.
3. ينشئ النظام تقريراً ائتمانياً شاملاً يحتوي على:
   - معلومات المقترض الشخصية/الشركة
   - التقييم الائتماني (نطاق 300-850)
   - رموز الأسباب التي تشرح التقييم
   - جميع الحسابات الائتمانية مع الحالة والأرصدة
   - سجل أداء المدفوعات
   - الاستعلامات الائتمانية
   - الأحكام القضائية
   - سجلات الموافقة
   - الرقم التسلسلي للتقرير

### 6.4 فهم التقييمات الائتمانية (300-850)

تقوم خوارزمية التقييم الائتماني بتعيين درجة بين 300 و850:

| نطاق الدرجة | التصنيف | الوصف |
|-------------|---------|-------|
| 750-850 | ممتاز | مخاطر ائتمانية منخفضة جداً |
| 700-749 | جيد | مخاطر ائتمانية منخفضة |
| 650-699 | مقبول | مخاطر ائتمانية معتدلة |
| 550-649 | ضعيف | مخاطر ائتمانية عالية |
| 300-549 | ضعيف جداً | مخاطر ائتمانية عالية جداً |

العوامل التي تؤثر على التقييم الائتماني تشمل:
- نسبة الدفع في الوقت المحدد
- عدد الحسابات المتأخرة
- الحسابات المشطوبة
- عدد الاستعلامات الأخيرة
- الأحكام القضائية
- مستويات الدين

### 6.5 شرح رموز الأسباب

تتضمن التقارير الائتمانية رموز أسباب تشرح العوامل الرئيسية المؤثرة على التقييم الائتماني للمقترض:

| رمز السبب | الوصف |
|-----------|-------|
| DELINQUENT_ACCOUNTS | لدى المقترض حساب متأخر واحد أو أكثر |
| WRITTEN_OFF_ACCOUNTS | لدى المقترض حسابات تم شطبها |
| RESTRUCTURED_ACCOUNTS | لدى المقترض تسهيلات قروض أعيدت هيكلتها |
| HIGH_INQUIRY_VOLUME | عدد مرتفع من الاستعلامات الائتمانية الأخيرة |
| HIGH_DEBT_LEVEL | مستوى الدين الإجمالي مرتفع |
| COURT_JUDGMENTS_PRESENT | توجد أحكام قضائية نشطة ضد المقترض |
| POLITICALLY_EXPOSED_PERSON | المقترض موسوم كشخص مكشوف سياسياً |
| STRONG_REPAYMENT_HISTORY | سجل سداد منتظم في الوقت المحدد (إيجابي) |
| EXCELLENT_PAYMENT_RECORD | سجل دفع متميز (إيجابي) |
| THIN_FILE_LIMITED_HISTORY | سجل ائتماني محدود متاح |

### 6.6 الأرقام التسلسلية للتقارير

يتم تعيين رقم تسلسلي فريد لكل تقرير ائتماني يتم إنشاؤه بالتنسيق:

```
CR-{YEAR}-{unique_identifier}
```

مثال: `CR-2025-k8f3x2m1`

يعمل هذا الرقم التسلسلي كمرجع للتقرير ويمكن استخدامه لأغراض التدقيق والتتبع.

### 6.7 طباعة التقارير

1. بعد إنشاء تقرير ائتماني، انقر على زر **طباعة**.
2. يُفتح التقرير بتنسيق ملائم للطباعة.
3. استخدم مربع حوار الطباعة في المتصفح للطباعة أو الحفظ كملف PDF.
4. يتضمن التقرير المطبوع جميع الأقسام والرقم التسلسلي والطابع الزمني للإنشاء وإخلاء مسؤولية في التذييل.

## 7. سير عمل صانع-مدقق

يضمن سير عمل صانع-مدقق (مبدأ العيون الأربع) سلامة البيانات من خلال اشتراط موافقة مستخدم مصرّح ثانٍ على التغييرات.

### 7.1 كيفية تقديم التغييرات

عند إنشاء أو تعديل مقترض أو حساب ائتماني:

1. لا يتم تطبيق تغييرك **فوراً**.
2. بدلاً من ذلك، يتم إنشاء **طلب موافقة معلق**.
3. سترى رسالة تأكيد: "تم الإرسال لموافقة صانع-مدقق."
4. يتلقى المراجعون المصرّحون (أدوار المسؤول والمنظم) إشعاراً.

### 7.2 مراجعة الموافقات المعلقة

1. انتقل إلى **الموافقات المعلقة** من الشريط الجانبي.
2. تعرض الصفحة جميع طلبات الموافقة المعلقة مع:
   - نوع الكيان (مقترض أو حساب ائتماني)
   - الإجراء (إنشاء أو تحديث)
   - اسم مقدم الطلب
   - تاريخ التقديم
   - الحالة الحالية
3. انقر على طلب معلق لمراجعة التفاصيل.

### 7.3 الموافقة/الرفض مع ملاحظات

1. افتح طلب موافقة معلق.
2. راجع البيانات المقدمة بعناية.
3. اختر أحد الإجراءات التالية:
   - **موافقة** — سيتم تطبيق التغيير على قاعدة البيانات.
   - **رفض** — سيتم تجاهل التغيير.
4. اختيارياً أدخل **ملاحظات المراجعة** لشرح قرارك.
5. انقر على الزر المناسب لتقديم قرارك.
6. سيتلقى مقدم الطلب إشعاراً بالقرار.

### 7.4 قاعدة منع الموافقة الذاتية

يفرض النظام قاعدة صارمة: **لا يمكنك الموافقة على طلباتك الخاصة**. هذا يعني:
- إذا قدمت تسجيل مقترض، يجب أن يوافق عليه مسؤول أو منظم مختلف.
- محاولة الموافقة على طلبك الخاص ستؤدي إلى رسالة خطأ.
- هذا يضمن الامتثال الحقيقي لمبدأ العيون الأربع.

## 8. إدارة النزاعات

تتعامل وحدة إدارة النزاعات مع شكاوى المقترضين وطلبات تصحيح البيانات.

### 8.1 تقديم نزاع

1. انتقل إلى **النزاعات** من الشريط الجانبي.
2. انقر على زر **تقديم نزاع**.
3. املأ الحقول المطلوبة:
   - **المقترض** — اختر المقترض من القائمة المنسدلة
   - **الحساب الائتماني** — اختيارياً اختر الحساب المحدد المعني
   - **نوع النزاع** — اختر من:
     - Data Error
     - Identity Theft
     - Unauthorized Inquiry
     - Duplicate Record
     - Other
   - **الوصف** — قدم وصفاً مفصلاً للمشكلة
   - **نوع التصحيح** — اختر:
     - Financial (اتفاقية مستوى الخدمة يومان)
     - Non-Financial (اتفاقية مستوى الخدمة 5 أيام)
4. انقر على **تقديم نزاع** للإرسال.

### 8.2 تتبع اتفاقية مستوى الخدمة (يومان مالي، 5 أيام غير مالي)

يقوم النظام تلقائياً بحساب وتتبع مواعيد اتفاقية مستوى الخدمة:

- **التصحيحات المالية**: يجب حلها خلال **يومي عمل** (DQ-04)
- **التصحيحات غير المالية**: يجب حلها خلال **5 أيام عمل** (DQ-05)

يتم عرض الموعد النهائي لاتفاقية مستوى الخدمة في جدول النزاعات. إذا انقضى الموعد النهائي والنزاع لا يزال مفتوحاً، تظهر شارة "مخترق" حمراء.

### 8.3 حل النزاعات

1. في جدول النزاعات، انقر على زر **حل** بجانب نزاع مفتوح (أو انقر على الصف لفتح مربع حوار الحل).
2. اختر الحالة الجديدة:
   - **قيد المراجعة** — التحقيق جارٍ
   - **تم الحل** — تم تصحيح المشكلة
   - **مرفوض** — تبين أن النزاع غير صالح
3. أدخل **ملاحظات الحل** التي تصف النتيجة.
4. انقر على **تحديث النزاع** للحفظ.

### 8.4 عرض سجل النزاعات

يعرض جدول النزاعات جميع النزاعات مع حالتها الحالية ونوعها والموعد النهائي لاتفاقية مستوى الخدمة وتاريخ التقديم. يمكنك مراجعة السجل الكامل للنزاعات بما في ذلك القرارات والطوابع الزمنية.

## 9. الأحكام القضائية

تتتبع وحدة الأحكام القضائية الإجراءات القانونية وأوامر المحكمة ضد المقترضين (FR-COL-05).

### 9.1 إضافة أحكام قضائية

1. انتقل إلى صفحة تفاصيل المقترض.
2. في قسم **الأحكام القضائية**، انقر على **إضافة حكم**.
3. املأ الحقول المطلوبة:
   - **رقم القضية** — رقم مرجع قضية المحكمة
   - **المحكمة** — اسم المحكمة
   - **نوع الحكم** — اختر من:
     - Lien
     - Bankruptcy
     - Lawsuit
     - Civil Judgment
     - Criminal Conviction
   - **المبلغ** — القيمة النقدية للحكم (إن وُجد)
   - **العملة** — عملة مبلغ الحكم
   - **تاريخ الحكم** — تاريخ إصدار الحكم
   - **الحالة** — Active أو Resolved أو Appealed
   - **الوصف** — تفاصيل إضافية حول الحكم
4. انقر على **إرسال** لإنشاء سجل الحكم.

### 9.2 عرض الأحكام حسب المقترض

تُعرض الأحكام القضائية في صفحة تفاصيل المقترض وتُدرج في التقارير الائتمانية. يعرض كل حكم:
- رقم القضية واسم المحكمة
- نوع الحكم وحالته
- المبلغ والعملة
- تاريخ الحكم

### 9.3 أنواع الأحكام

| النوع | الوصف |
|-------|-------|
| **Lien** | مطالبة قانونية على ممتلكات كضمان لدين |
| **Bankruptcy** | إعلان رسمي بعدم القدرة على سداد الديون |
| **Lawsuit** | دعوى قانونية مدنية قائمة أو منتهية |
| **Civil Judgment** | أمر محكمة في قضية مدنية (مثل استرداد الديون) |
| **Criminal Conviction** | حكم إدانة جنائي ذو صلة بالأمور المالية |

## 10. إدارة الموافقات

تدير وحدة إدارة الموافقات موافقة أصحاب البيانات على مشاركة المعلومات الائتمانية (FR-CON-06/07).

### 10.1 منح الموافقة

1. انتقل إلى **إدارة الموافقات** من الشريط الجانبي.
2. انقر على زر **منح الموافقة**.
3. املأ الحقول المطلوبة:
   - **المقترض** — اختر المقترض الذي يمنح الموافقة
   - **ممنوحة لـ** — اسم المؤسسة أو الجهة المتلقية للموافقة
   - **الغرض** — سبب مشاركة البيانات (مثل تقييم ائتماني، مراجعة تنظيمية)
   - **نوع الموافقة** — نوع الموافقة (مثل استعلام، مشاركة بيانات، وصول كامل)
4. انقر على **منح الموافقة** لإنشاء سجل الموافقة.

### 10.2 أرقام الإيصالات

يتم تعيين رقم إيصال فريد تلقائياً لكل سجل موافقة بالتنسيق:

```
CR-{timestamp}-{random_identifier}
```

يعمل رقم الإيصال هذا كإثبات للموافقة ويمكن تقديمه للمقترض أو المؤسسة الطالبة.

### 10.3 إلغاء الموافقة

1. في جدول سجلات الموافقة، ابحث عن سجل الموافقة النشط.
2. انقر على زر **إلغاء**.
3. سيتم تغيير حالة الموافقة إلى "ملغاة" وسيتم تسجيل طابع زمني للإلغاء.
4. بمجرد الإلغاء، لم يعد بإمكان المؤسسة الوصول إلى بيانات ائتمان المقترض بموجب هذه الموافقة.

### 10.4 عرض سجلات الموافقة

تعرض صفحة إدارة الموافقات جميع سجلات الموافقة مع:
- اسم المقترض ورقم هويته
- المؤسسة الممنوح لها الموافقة
- الغرض ونوع الموافقة
- الحالة (نشطة أو ملغاة)
- رقم الإيصال
- تاريخ المنح وتاريخ الإلغاء (إن وُجد)

## 11. إدارة المؤسسات (المسؤول فقط)

وحدة إدارة المؤسسات متاحة فقط للمستخدمين ذوي دور **المسؤول**.

### 11.1 تسجيل المؤسسات

1. انتقل إلى **المؤسسات** من الشريط الجانبي.
2. انقر على زر **تسجيل**.
3. املأ الحقول المطلوبة:
   - **الاسم** — اسم المؤسسة
   - **النوع** — اختر من:
     - Bank
     - MFI (مؤسسة تمويل أصغر)
     - Utility
     - Telecom
     - Digital Lender
     - SACCO (جمعية ادخار وائتمان تعاونية)
   - **رقم التسجيل** — رقم التسجيل/الترخيص الرسمي
   - **البلد** — بلد العمل
   - **البريد الإلكتروني للتواصل** — عنوان البريد الإلكتروني الرئيسي
   - **هاتف التواصل** — رقم الهاتف الرئيسي
   - **العنوان** — العنوان الفعلي
   - **تكرار التقديم** — كم مرة تقدم المؤسسة بياناتها:
     - Daily
     - Weekly
     - Monthly
4. انقر على **تسجيل** لتقديم المؤسسة للموافقة.

### 11.2 الموافقة على المؤسسات

المؤسسات المسجلة حديثاً لها حالة "معلق" وتتطلب موافقة المسؤول:

1. في جدول المؤسسات، حدد المؤسسات ذات حالة "معلق".
2. انقر على زر **موافقة**.
3. ستتغير حالة المؤسسة إلى "نشطة".
4. يمكن بعد ذلك تعيين مفاتيح API للمؤسسات النشطة لتقديم البيانات.

### 11.3 تهيئة تكرار التقديم

يحدد تكرار التقديم عدد مرات توقع تقديم المؤسسة لبيانات ائتمانية. يمكن ضبطه أثناء التسجيل ويُعرض في تفاصيل المؤسسة.

### 11.4 أنواع المؤسسات

| النوع | الوصف |
|-------|-------|
| **Bank** | بنك تجاري أو تجزئة |
| **MFI** | مؤسسة تمويل أصغر |
| **Utility** | مزود خدمات مرافق |
| **Telecom** | شركة اتصالات |
| **Digital Lender** | منصة إقراض رقمية/محمولة |
| **SACCO** | منظمة ادخار وائتمان تعاونية |

### 11.5 التصفح بالصفحات

تدعم قائمة المؤسسات التصفح بالصفحات لمجموعات البيانات الكبيرة:
- استخدم زري **السابق** و**التالي** للتنقل بين الصفحات.
- يتم عرض 50 سجلاً لكل صفحة.

## 12. الفوترة (المسؤول/المنظم)

تدير وحدة الفوترة الفواتير وتتبع الرسوم لمؤسسات مزودي البيانات. وهي متاحة للمستخدمين ذوي أدوار **المسؤول** أو **المنظم**.

### 12.1 إنشاء الفواتير

1. انتقل إلى **الفوترة** من الشريط الجانبي.
2. انقر على زر **إنشاء فاتورة**.
3. املأ الحقول المطلوبة:
   - **اسم المؤسسة** — اسم المؤسسة المفوترة
   - **نوع الخدمة** — اختر من:
     - Data Submission
     - Credit Report
     - API Access
     - Subscription
   - **المبلغ** — مبلغ الفاتورة
   - **العملة** — العملة (ETB، USD، KES، GHS، UGX)
   - **رقم الفاتورة** — رقم مرجع فاتورة فريد
   - **بداية الفترة** — تاريخ بدء فترة الفوترة
   - **نهاية الفترة** — تاريخ انتهاء فترة الفوترة
4. انقر على **إنشاء فاتورة** للحفظ.

### 12.2 تتبع حالة الدفع

تعرض صفحة الفوترة ثلاث بطاقات ملخص:
- **إجمالي الإيرادات** — مجموع جميع مبالغ الفواتير
- **المبلغ المعلق** — مجموع الفواتير غير المدفوعة
- **المبلغ المتأخر** — مجموع الفواتير المتأخرة

حالات الفواتير:
| الحالة | الوصف |
|--------|-------|
| **Pending** | تم إصدار الفاتورة ولكن لم يتم الدفع بعد |
| **Paid** | تم دفع الفاتورة بالكامل |
| **Overdue** | تجاوزت الفاتورة تاريخ استحقاقها ولم تُدفع |

### 12.3 أنواع الخدمات

| نوع الخدمة | الوصف |
|------------|-------|
| **Data Submission** | رسوم تقديم البيانات الائتمانية للسجل |
| **Credit Report** | رسوم إنشاء التقارير الائتمانية |
| **API Access** | رسوم استخدام API الخارجي |
| **Subscription** | رسوم اشتراك دورية |

### 12.4 عرض تفاصيل الفاتورة

انقر على أي صف في جدول الفوترة لفتح عرض مفصل يُظهر:
- رقم الفاتورة وحالتها
- اسم المؤسسة
- نوع الخدمة
- المبلغ والعملة
- تواريخ فترة الفوترة
- تاريخ الإنشاء

## 13. مكتب المساعدة

يوفر مكتب المساعدة (وحدة خدمة الاستعلامات) بوابة موحدة لعمليات خدمة المستهلك.

### 13.1 البحث عن المقترضين

1. انتقل إلى **مكتب المساعدة** من الشريط الجانبي.
2. راجع بطاقات الملخص التي تعرض:
   - عدد الاستعلامات المفتوحة
   - عدد خروقات اتفاقية مستوى الخدمة
   - عدد المحلولة اليوم
3. أدخل اسم المقترض أو رقم هويته الوطنية في حقل البحث.
4. تظهر النتائج كبطاقات قابلة للنقر أسفل حقل البحث.
5. انقر على بطاقة مقترض لاختياره وعرض تفاصيله.

### 13.2 عرض معلومات المقترض والنزاعات والموافقات

بمجرد اختيار مقترض، يعرض مكتب المساعدة:
- **معلومات المقترض** — رقم الهوية الوطنية، الهاتف، البريد الإلكتروني، النوع
- **النزاعات** — جميع النزاعات المرتبطة بالمقترض، بما في ذلك النوع والوصف والحالة والموعد النهائي لاتفاقية مستوى الخدمة وتاريخ التقديم
- **سجلات الموافقة** — جميع سجلات الموافقة للمقترض، تُظهر المؤسسة الممنوح لها والغرض ونوع الموافقة والحالة ورقم الإيصال وتاريخ المنح

### 13.3 تقديم النزاعات من مكتب المساعدة

1. مع اختيار مقترض، انقر على زر **تقديم نزاع**.
2. يتم ملء اسم المقترض مسبقاً.
3. املأ:
   - **معرّف الحساب الائتماني** (اختياري)
   - **نوع النزاع** — Incorrect Balance، Wrong Status، Identity Error، Unauthorized Inquiry، Other
   - **نوع التصحيح** — Financial أو Non-Financial
   - **الوصف** — وصف مفصل للمشكلة
4. انقر على **تقديم نزاع** للإرسال.

### 13.4 منح الموافقة من مكتب المساعدة

1. مع اختيار مقترض، انقر على زر **منح الموافقة**.
2. املأ:
   - **ممنوحة لـ** — اسم المؤسسة أو الجهة
   - **الغرض** — سبب مشاركة البيانات
   - **نوع الموافقة** — نوع الموافقة الممنوحة
3. انقر على **منح الموافقة** لإنشاء السجل.
4. يتم إنشاء رقم إيصال وتعيينه تلقائياً.

## 14. التحميل الجماعي

تتيح وحدة التحميل الجماعي استيعاب البيانات بالجملة من خلال تحميل ملفات JSON أو CSV.

### 14.1 إعداد بيانات JSON

أنشئ ملف JSON يحتوي على مصفوفة من سجلات الحسابات الائتمانية. يجب أن يتضمن كل سجل:

```json
[
  {
    "borrowerId": "borrower-uuid",
    "lenderInstitution": "Bank Name",
    "accountNumber": "ACC-001",
    "accountType": "Personal Loan",
    "originalAmount": "50000.00",
    "currentBalance": "35000.00",
    "currency": "ETB",
    "interestRate": "12.50",
    "status": "current"
  }
]
```

### 14.2 إعداد بيانات CSV

أنشئ ملف CSV برؤوس تطابق الحقول المطلوبة:

```csv
borrowerId,lenderInstitution,accountNumber,accountType,originalAmount,currentBalance,currency,interestRate,status
uuid-1,Bank A,ACC-001,Personal Loan,50000.00,35000.00,ETB,12.50,current
uuid-2,Bank B,ACC-002,Mortgage,200000.00,180000.00,ETB,9.00,current
```

### 14.3 تحميل الملفات

1. انتقل إلى **التحميل الجماعي** من الشريط الجانبي.
2. اختر نوع الملف (JSON أو CSV).
3. انقر على زر **تحميل** أو اسحب وأفلت ملفك في منطقة التحميل.
4. يعالج النظام الملف ويتحقق من صحة كل سجل.

### 14.4 مراجعة نتائج التحقق

بعد التحميل، يعرض النظام:
- **إجمالي السجلات** — عدد السجلات في الملف
- **ناجحة** — السجلات التي اجتازت التحقق وتم استيرادها
- **فاشلة** — السجلات التي بها أخطاء تحقق

### 14.5 تنزيل تقارير الأخطاء

إذا فشل أي سجل في التحقق:
1. راجع تفاصيل الأخطاء التي تُظهر السجلات التي فشلت وسبب الفشل.
2. يتضمن كل خطأ رقم السجل واسم الحقل ووصف الخطأ.
3. صحح الأخطاء في ملفك المصدر وأعد التحميل.

## 15. سجل التدقيق (المسؤول/المنظم)

يوفر سجل التدقيق سجلاً غير قابل للتغيير لجميع أنشطة النظام. وهو متاح للمستخدمين ذوي أدوار **المسؤول** أو **المنظم**.

### 15.1 عرض نشاط النظام

1. انتقل إلى **سجل التدقيق** من الشريط الجانبي.
2. يعرض سجل التدقيق إدخالات مع:
   - **الطابع الزمني** — متى حدث الإجراء
   - **المستخدم** — من نفذ الإجراء
   - **الإجراء** — نوع الإجراء المنفذ
   - **الكيان** — نوع الكيان المتأثر
   - **معرّف الكيان** — معرّف السجل المتأثر
   - **التفاصيل** — وصف ما حدث
   - **عنوان IP** — عنوان IP الخاص بالمستخدم

### 15.2 فهم أنواع الإجراءات

| الإجراء | الوصف |
|---------|-------|
| LOGIN | قام المستخدم بتسجيل الدخول إلى النظام |
| LOGOUT | قام المستخدم بتسجيل الخروج |
| LOGIN_FAILED | محاولة تسجيل دخول فاشلة |
| ACCOUNT_LOCKED | تم قفل الحساب بعد محاولات فاشلة |
| CREATE | تم إنشاء سجل جديد |
| UPDATE | تم تعديل سجل قائم |
| VIEW | تم عرض/الوصول إلى سجل |
| SUBMIT_APPROVAL | تم تقديم تغيير لموافقة صانع-مدقق |
| APPROVE | تمت الموافقة على طلب معلق |
| REJECT | تم رفض طلب معلق |
| FILE_DISPUTE | تم تقديم نزاع جديد |
| RESOLVE_DISPUTE | تم حل النزاع |
| PASSWORD_CHANGE | قام المستخدم بتغيير كلمة المرور |

### 15.3 تصفية الإدخالات

يمكن تصفية سجل التدقيق والبحث فيه حسب:
- نطاق التاريخ
- المستخدم
- نوع الإجراء
- نوع الكيان
- عنوان IP

## 16. إدارة المستخدمين (المسؤول فقط)

وحدة إدارة المستخدمين متاحة فقط للمستخدمين ذوي دور **المسؤول**.

### 16.1 إنشاء المستخدمين

1. انتقل إلى **إدارة المستخدمين** من الشريط الجانبي.
2. انقر على زر **إضافة مستخدم**.
3. املأ الحقول المطلوبة:
   - **اسم المستخدم** — اسم مستخدم فريد لتسجيل الدخول
   - **كلمة المرور** — كلمة مرور أولية (يجب أن تستوفي متطلبات التعقيد)
   - **الاسم الكامل** — الاسم الكامل للمستخدم
   - **البريد الإلكتروني** — عنوان البريد الإلكتروني للمستخدم
   - **الدور** — اختر من Admin أو Regulator أو Lender أو Viewer
   - **المؤسسة** — اسم المؤسسة المرتبطة (اختياري)
4. انقر على **إنشاء مستخدم** للحفظ.

### 16.2 تعيين الأدوار

يتم تعيين أحد الأدوار الأربعة لكل مستخدم:

| الدور | الوصف |
|-------|-------|
| **Admin** | وصول كامل للنظام بما في ذلك إدارة المستخدمين والمؤسسات ومفاتيح API |
| **Regulator** | وصول إلى معظم الوحدات بما في ذلك الموافقات وسجل التدقيق والفوترة |
| **Lender** | وصول إلى المقترضين والحسابات الائتمانية والنزاعات والتحميل الجماعي |
| **Viewer** | وصول للقراءة فقط إلى المقترضين والحسابات الائتمانية والتقارير |

### 16.3 تغيير حالة المستخدم

يمكن أن يكون لحسابات المستخدمين إحدى ثلاث حالات:

| الحالة | الوصف |
|--------|-------|
| **Active** | يمكن للمستخدم تسجيل الدخول والوصول إلى النظام بشكل طبيعي |
| **Suspended** | تم حظر المستخدم مؤقتاً من تسجيل الدخول |
| **Deactivated** | تم حظر المستخدم نهائياً من تسجيل الدخول |

لتغيير حالة المستخدم:
1. في جدول إدارة المستخدمين، انقر على إجراء **تعديل** للمستخدم المستهدف.
2. حدّث حقل الحالة.
3. انقر على **حفظ** لتطبيق التغيير.

### 16.4 فهم صلاحيات الأدوار

راجع **الملحق ب: مصفوفة صلاحيات الأدوار** للحصول على تفصيل كامل لما يمكن لكل دور الوصول إليه.

## 17. مفاتيح API (المسؤول فقط)

تتيح وحدة مفاتيح API للمسؤولين إدارة الوصول الخارجي لـ API للمؤسسات.

### 17.1 إنشاء مفاتيح API للمؤسسات

1. انتقل إلى **مفاتيح API** من الشريط الجانبي.
2. انقر على زر **إنشاء مفتاح**.
3. املأ الحقول المطلوبة:
   - **المؤسسة** — اختر مؤسسة نشطة من القائمة المنسدلة
   - **التسمية** — اسم وصفي للمفتاح (مثل "مفتاح الإنتاج")
   - **الصلاحيات** — اختر مستوى الصلاحية:
     - **Submit** — يمكنه تقديم المقترضين والحسابات وبيانات الدفع
     - **Read** — يمكنه البحث عن المقترضين واسترجاع التقارير الائتمانية
     - **Full** — صلاحيات التقديم والقراءة معاً
4. انقر على **إنشاء مفتاح**.
5. سيُعرض مفتاح API المُنشأ **مرة واحدة فقط**. انسخه فوراً.

### 17.2 فهم مستويات الصلاحيات

| الصلاحية | الإمكانيات |
|----------|-----------|
| **Submit** | POST المقترضون، الحسابات الائتمانية، سجل المدفوعات، الأحكام القضائية |
| **Read** | GET البحث عن المقترضين، التقارير الائتمانية، الحسابات الائتمانية |
| **Full** | جميع إمكانيات التقديم والقراءة مجتمعة |

### 17.3 إلغاء مفتاح API

1. في جدول مفاتيح API، ابحث عن المفتاح الذي تريد إلغاءه.
2. انقر على زر **إلغاء**.
3. ستتغير حالة المفتاح إلى "ملغى" ولن يمكن استخدامه بعد ذلك لمصادقة API.
4. الإلغاء غير قابل للتراجع؛ يجب إنشاء مفتاح جديد إذا كانت هناك حاجة للوصول مجدداً.

### 17.4 عرض الاستخدام

يعرض جدول مفاتيح API:
- **بادئة المفتاح** — أول أحرف من المفتاح للتعريف
- **التسمية** — الاسم الوصفي
- **المؤسسة** — المؤسسة المرتبطة
- **الصلاحيات** — مستوى الصلاحية
- **الحالة** — نشط أو ملغى
- **آخر استخدام** — آخر مرة تم فيها استخدام المفتاح لمصادقة API
- **تاريخ الإنشاء** — متى تم إنشاء المفتاح

## 18. التقارير والتصدير

توفر وحدة التقارير عروضاً تحليلية وإمكانيات تصدير البيانات.

### 18.1 تصدير CSV (المحفظة/المقترضون)

1. انتقل إلى **التقارير** من الشريط الجانبي (عبر التقارير الائتمانية).
2. اختر نوع التصدير:
   - **تصدير المحفظة** — جميع الحسابات الائتمانية مع الأرصدة والحالات

3. انقر على **تصدير CSV**.
4. سيتم تنزيل ملف CSV إلى حاسوبك.

### 18.2 عرض التحليلات التنظيمية

يوفر قسم التحليلات التنظيمية:
- **نسب NPL** — نسب القروض غير العاملة حسب المؤسسة ونوع القرض
- **تقسيمات المحفظة** — توزيع الائتمان حسب النوع والحالة والعملة
- **تتبع خروقات اتفاقية مستوى الخدمة** — النزاعات التي تجاوزت مواعيد اتفاقية مستوى الخدمة
- **امتثال تقديم البيانات** — المؤسسات التي تستوفي متطلبات تكرار التقديم

## 20. الأسئلة الشائعة / "كيف أقوم بـ...؟"

### س: كيف أعيد تعيين كلمة المرور؟
ج: انقر على خيار تغيير كلمة المرور في الشريط الجانبي، أدخل كلمة المرور الحالية وكلمة المرور الجديدة، ثم انقر على تغيير كلمة المرور. إذا نسيت كلمة المرور، تواصل مع مسؤول النظام.

### س: كيف أسجل مقترضاً جديداً؟
ج: انتقل إلى المقترضون، انقر على "تسجيل مقترض"، املأ النموذج، وأرسل. سيمر طلبك عبر موافقة صانع-مدقق قبل إنشاء المقترض.

### س: لماذا لا أستطيع الموافقة على تقديمي الخاص؟
ج: يفرض النظام مبدأ صانع-مدقق (العيون الأربع). يجب على مستخدم مصرّح مختلف الموافقة على تقديماتك لضمان سلامة البيانات.

### س: ماذا يحدث عند انتهاء مهلة جلستي؟
ج: بعد 15 دقيقة من عدم النشاط، يتم تسجيل خروجك تلقائياً. انتقل إلى صفحة تسجيل الدخول وسجل الدخول مرة أخرى. سيتم فقدان أي تغييرات غير محفوظة.

### س: كيف أبحث عن مقترض؟
ج: استخدم شريط البحث في صفحة المقترضين. يمكنك البحث بالاسم أو رقم الهوية الوطنية أو معلومات تعريفية أخرى.

### س: ما هي علامة PEP؟
ج: PEP تعني الشخص المكشوف سياسياً. تشير هذه العلامة إلى أن المقترض يشغل أو شغل منصباً عاماً بارزاً.

### س: كم من الوقت لدي لحل نزاع؟
ج: يجب حل التصحيحات المالية خلال يومي عمل. يجب حل التصحيحات غير المالية خلال 5 أيام عمل.

### س: كيف أنشئ تقريراً ائتمانياً؟
ج: انتقل إلى صفحة تفاصيل المقترض وانقر على "إنشاء تقرير ائتماني"، أو استخدم صفحة البحث الائتماني للبحث عن مقترض ثم إنشاء التقرير.

### س: هل يمكنني تصدير بيانات من النظام؟
ج: نعم. انتقل إلى التقارير واستخدم ميزة تصدير CSV لتنزيل بيانات المحفظة أو المقترضين.

### س: كيف أبدل اللغات؟
ج: انقر على زر تبديل اللغة (EN/FR/PT) في شريط العنوان العلوي أو في صفحة تسجيل الدخول. ستتبدل الواجهة فوراً.

### س: كيف أبدل بين الوضع الفاتح والداكن؟
ج: انقر على أيقونة تبديل السمة (شمس/قمر) في شريط العنوان.

### س: كيف أمنح الموافقة لمقترض؟
ج: انتقل إلى إدارة الموافقات وانقر على "منح الموافقة"، أو استخدم وحدة مكتب المساعدة لمنح الموافقة لمقترض مختار.

### س: ما هي مفاتيح API المستخدمة لها؟
ج: تمكّن مفاتيح API المؤسسات الخارجية من تقديم البيانات برمجياً واسترجاع التقارير الائتمانية عبر API الخارجي.

### س: كيف أحمّل بيانات بالجملة؟
ج: استخدم صفحة التحميل الجماعي. جهّز بياناتك بتنسيق JSON أو CSV أو XBRL وحمّل الملف. يتحقق النظام من صحة كل سجل ويُبلغ عن أي أخطاء.

### س: كيف أمكّن MFA؟
ج: اذهب إلى إعدادات ملفك الشخصي وانقر على "تمكين MFA". سيُعرض لك رمز QR أو URI إعداد لمسحه بتطبيق المصادقة. أدخل الرمز المكون من 6 أرقام للتحقق وتفعيل MFA.

### س: كيف أتحقق من سلامة سجل التدقيق؟
ج: انتقل إلى صفحة سجل التدقيق وانقر على زر "التحقق من السلامة". سيتحقق النظام من سلسلة تجزئة SHA-256 ويعرض ما إذا كانت السجلات سليمة.

### س: ما هو روبوت النزاعات؟
ج: روبوت النزاعات هو مساعد موجه متاح في صفحة مكتب المساعدة. يرشدك خلال تقديم نزاع خطوة بخطوة: اختيار نوع المشكلة، العثور على المقترض، اختيار الحساب، إدخال الوصف، والإرسال.

### س: هل يمكنني استخدام رموز OAuth بدلاً من مفاتيح API؟
ج: نعم. يمكنك استبدال مفتاح API الخاص بك برمز Bearer باستخدام نقطة نهاية رمز OAuth 2.1. الرمز صالح لمدة ساعة واحدة ويمكن استخدامه في ترويسة `Authorization: Bearer`.

### س: من يمكنه الموافقة على الطلبات المعلقة؟
ج: المستخدمون ذوو دور Admin أو Regulator يمكنهم الموافقة على الطلبات المعلقة أو رفضها، طالما لم يقدموا الطلب بأنفسهم.

### س: كيف أضيف حكماً قضائياً؟
ج: انتقل إلى صفحة تفاصيل المقترض، ابحث عن قسم الأحكام القضائية، وانقر على "إضافة حكم". فقط مستخدمو Admin وRegulator يمكنهم إنشاء أحكام قضائية.

### س: ما العملات التي يدعمها النظام؟
ج: يدعم النظام أكثر من 42 عملة أفريقية بالإضافة إلى USD وEUR وGBP (أكثر من 45 إجمالاً). راجع الملحق ج للحصول على القائمة الكاملة.

### س: كيف أستخدم الجولة التوضيحية التفاعلية؟
ج: تنطلق الجولة الإرشادية التفاعلية تلقائياً بعد تسجيل الدخول عبر بيئة العرض التوضيحي. تُرشدك عبر 11 ميزة رئيسية في النظام مع طبقة تسليط ضوئي وتلميحات وصفية. استخدم أزرار Next وBack وSkip وClose للتنقل في الجولة. إذا أردت إعادة تشغيل الجولة بعد انتهائها، انقر على زر "Take a Tour" في شعار العرض التوضيحي الكهرماني أعلى الصفحة.

## 21. المصادقة متعددة العوامل (MFA)

> يغطي هذا القسم ENT-01. للتحسينات المؤسسية الأخرى، راجع: المطابقة الضبابية (§21.4)، روبوت النزاعات (§22)، تحميل XBRL (§23)، سلامة التدقيق (§24)، OAuth 2.1 (§21.5)، النطاق الترددي المنخفض (§21.6).

يدعم النظام المصادقة متعددة العوامل المبنية على TOTP لتعزيز أمان تسجيل الدخول.

### 21.1 تمكين MFA

1. سجل الدخول إلى النظام بشكل طبيعي.
2. انتقل إلى ملفك الشخصي أو إعدادات MFA (يمكن الوصول إليها من الشريط الجانبي أو قائمة المستخدم).
3. انقر على **تمكين MFA** أو **إعداد MFA**.
4. ينشئ النظام سر TOTP ويعرض URI بتنسيق `otpauth://`.
5. امسح رمز QR أو أدخل السر يدوياً في تطبيق المصادقة (Google Authenticator أو Authy أو Microsoft Authenticator أو غيرها).
6. أدخل الرمز المكون من 6 أرقام الذي يعرضه تطبيق المصادقة.
7. انقر على **تحقق** لتفعيل MFA.
8. تؤكد رسالة نجاح أنه تم تمكين MFA الآن.

### 21.2 تسجيل الدخول مع MFA

1. أدخل اسم المستخدم وكلمة المرور في صفحة تسجيل الدخول.
2. انقر على **تسجيل الدخول**.
3. إذا كان MFA مُمكّناً، تظهر شاشة ثانية تطلب رمز TOTP.
4. افتح تطبيق المصادقة وأدخل الرمز الحالي المكون من 6 أرقام.
5. انقر على **تحقق** لإكمال تسجيل الدخول.

### 21.3 تعطيل MFA

1. سجل الدخول إلى النظام.
2. انتقل إلى إعدادات MFA.
3. انقر على **تعطيل MFA**.
4. يتم تعطيل MFA فوراً. ستتطلب عمليات تسجيل الدخول المستقبلية اسم المستخدم وكلمة المرور فقط.

### 21.4 المطابقة الضبابية للكيانات (ENT-02)

عند تسجيل مقترض جديد، يتحقق النظام تلقائياً من التكرارات المحتملة باستخدام مطابقة الأسماء الضبابية. إذا تم العثور على مقترضين مشابهين:

1. يظهر **شريط تحذير** في نموذج التسجيل يسرد المقترضين ذوي الأسماء المشابهة ودرجات التشابه.
2. راجع المطابقات المدرجة لتحديد ما إذا كان المقترض موجوداً بالفعل في النظام.
3. إذا كان المقترض جديداً فعلاً، تابع التسجيل. إذا كان مكرراً، ألغِ واستخدم السجل الموجود.

تساعد هذه الميزة في منع السجلات المكررة والاحتيال المحتمل في الهوية عبر الولايات القضائية.

### 21.5 رموز Bearer لـ OAuth 2.1 (ENT-04)

للمؤسسات التي تستخدم API الخارجي، توفر رموز Bearer لـ OAuth 2.1 بديلاً لمصادقة مفتاح API:

1. انتقل إلى صفحة **وثائق API** من الشريط الجانبي.
2. انتقل إلى قسم **OAuth 2.1** للحصول على تعليمات تبادل الرموز ونماذج الكود.
3. استخدم `POST /api/external/oauth/token` مع بيانات اعتماد مفتاح API للحصول على رمز Bearer.
4. ضمّن الرمز في ترويسة `Authorization: Bearer <token>` في طلبات API.
5. تنتهي صلاحية الرموز بعد **ساعة واحدة** ويجب تحديثها بطلب رمز جديد.

### 21.6 تحسينات النطاق الترددي المنخفض (ENT-05)

يتضمن النظام تحسينات مدمجة للمستخدمين على الشبكات البطيئة أو المقيدة:

- **استجابات مضغوطة** — جميع استجابات الخادم مضغوطة بتنسيق gzip، مما يقلل نقل البيانات.
- **صفحات محملة بالطلب** — يتم تحميل صفحات التنقل عند الطلب بدلاً من تحميلها جميعاً دفعة واحدة، مما يؤدي إلى ظهور مؤشر تحميل مختصر عند زيارة صفحة لأول مرة.
- لا يلزم أي إجراء من المستخدم؛ هذه التحسينات نشطة دائماً.

## 22. روبوت النزاعات

يوفر روبوت النزاعات واجهة موجهة بأسلوب المحادثة لتقديم النزاعات.

### 22.1 الوصول إلى الروبوت

1. انتقل إلى **مكتب المساعدة** من الشريط الجانبي.
2. ابحث عن واجهة الروبوت أو الزر في صفحة مكتب المساعدة.

### 22.2 استخدام الروبوت

يرشدك الروبوت خلال تقديم النزاع في الخطوات التالية:

1. **اختيار نوع المشكلة** — اختر نوع النزاع (مثل Data Error، Identity Theft، Unauthorized Inquiry، Duplicate Record، Other).
2. **العثور على المقترض** — ابحث عن المقترض المتأثر بالاسم أو رقم الهوية الوطنية.
3. **اختيار الحساب** — اختيارياً اختر الحساب الائتماني المحدد المتعلق بالنزاع.
4. **وصف المشكلة** — أدخل وصفاً مفصلاً للمشكلة.
5. **التأكيد والإرسال** — راجع الملخص وأكد لتقديم النزاع تلقائياً.

### 22.3 الإلغاء

يمكنك إلغاء تدفق الروبوت في أي وقت. ستتم إعادة تعيين الواجهة ويمكنك بدء تدفق موجه جديد.

## 23. تحميل XBRL

تدعم وحدة التحميل الجماعي تنسيق ملف XBRL/XML بالإضافة إلى JSON وCSV.

### 23.1 الوصول إلى تحميل XBRL

1. انتقل إلى **التحميل الجماعي** من الشريط الجانبي.
2. انقر على تبويب **XBRL** (بجانب تبويب JSON/CSV).

### 23.2 إعداد بيانات XBRL

يعرض تبويب XBRL نموذج تنسيق XML يُظهر البنية المتوقعة لسجلات الحسابات الائتمانية. استخدم هذا النموذج كقالب عند إعداد ملف XBRL الخاص بك.

### 23.3 تحميل ملفات XBRL

1. في تبويب XBRL، انقر على منطقة التحميل أو اسحب وأفلت ملف `.xbrl` أو `.xml`.
2. يقوم النظام بتحليل XML والتحقق من صحة كل سجل.
3. تُعرض النتائج مع عمليات الاستيراد الناجحة وأي أخطاء تحقق.

## 24. التحقق من سلامة سجل التدقيق

يتضمن سجل التدقيق سلسلة تجزئة مقاومة للتلاعب توفر إثباتاً تشفيرياً بأنه لم يتم تعديل أو حذف أي إدخالات في السجل.

### 24.1 التحقق من السلامة

1. انتقل إلى **سجل التدقيق** من الشريط الجانبي (يتطلب وصول Admin أو Regulator).
2. ابحث عن شارة السلامة في أعلى الصفحة، تُظهر "صالح" (أخضر) أو "معطل" (أحمر).
3. انقر على زر **التحقق من السلامة** لإجراء تحقق جديد.

### 24.2 فهم النتائج

- **صالح** — جميع إدخالات سجل التدقيق تجتاز التحقق من سلسلة التجزئة. لم يتم اكتشاف تلاعب.
- **معطل** — سلسلة التجزئة معطلة عند إدخال محدد، مما يشير إلى تلاعب محتمل أو تلف في البيانات.

### 24.3 كيف يعمل

يحتوي كل إدخال في سجل التدقيق على تجزئة SHA-256 محسوبة من:
- تجزئة الإدخال السابق (مما ينشئ سلسلة)
- الإجراء والكيان والتفاصيل والطابع الزمني للإدخال الحالي

إذا تم تعديل أي إدخال أو حذفه أو إدراجه بترتيب خاطئ، تنقطع سلسلة التجزئة ويفشل فحص السلامة.

## 25. إدارة أسعار الصرف

تتيح وحدة إدارة أسعار الصرف للمسؤولين تهيئة وإدارة أسعار صرف العملات المستخدمة في جميع أنحاء النظام.

### 25.1 عرض أسعار الصرف

1. انتقل إلى **أسعار الصرف** من قسم الإدارة في الشريط الجانبي (يتطلب وصول Admin).
2. تعرض الصفحة جدولاً بجميع أسعار الصرف المهيأة، بما في ذلك:
   - **العملة الأساسية** — العملة المصدر
   - **العملة المستهدفة** — العملة الوجهة
   - **السعر** — سعر الصرف الحالي
   - **تاريخ السريان** — متى أصبح السعر سارياً
3. استخدم عناصر التحكم في البحث أو التصفية لتحديد أزواج عملات معينة.

### 25.2 إضافة سعر صرف جديد

1. انقر على زر **إضافة سعر** في أعلى صفحة أسعار الصرف.
2. املأ الحقول المطلوبة:
   - **العملة الأساسية** — اختر العملة المصدر من القائمة المنسدلة
   - **العملة المستهدفة** — اختر العملة الوجهة من القائمة المنسدلة
   - **السعر** — أدخل قيمة سعر الصرف
3. انقر على **حفظ** لإنشاء سجل سعر الصرف.

### 25.3 تعديل سعر صرف

1. حدد سعر الصرف الذي تريد تحديثه في الجدول.
2. انقر على زر **تعديل** (أيقونة القلم) في الصف المقابل.
3. عدّل قيمة السعر حسب الحاجة.
4. انقر على **حفظ** لتطبيق التغييرات.

### 25.4 حذف سعر صرف

1. حدد سعر الصرف الذي تريد إزالته في الجدول.
2. انقر على زر **حذف** (أيقونة سلة المهملات) في الصف المقابل.
3. أكد الحذف عند المطالبة.
4. سيتم إزالة سجل سعر الصرف نهائياً.

### 25.5 أداة تحويل العملات

تتضمن صفحة أسعار الصرف أداة تحويل عملات مدمجة:

1. في قسم المحول، اختر عملة **من**.
2. اختر عملة **إلى**.
3. أدخل **المبلغ** الذي تريد تحويله.
4. يُعرض المبلغ المحول تلقائياً بناءً على أسعار الصرف المهيأة.
5. إذا لم يكن هناك سعر صرف لزوج العملات المختار، ستظهر رسالة تشير إلى أن السعر غير متاح.

## 26. إدارة API

تتيح وحدة إدارة API للمسؤولين تهيئة وإدارة اتصالات API الخارجية المستخدمة من قبل النظام للتكامل مع خدمات الطرف الثالث.

### 26.1 الوصول إلى إدارة API

1. انتقل إلى **إدارة API** من قسم الإدارة في الشريط الجانبي (يتطلب وصول Admin).
2. تعرض الصفحة جميع اتصالات API المهيأة منظمة حسب الفئة.

### 26.2 فئات API

يتم تنظيم واجهات API الخارجية في الفئات التالية:

- **Weather** — خدمات بيانات الطقس للمعلومات الإقليمية
- **Judicial** — خدمات البحث عن السجلات القضائية والقانونية
- **Payment Gateway** — تكاملات معالجة المدفوعات
- **Other** — تكاملات خدمات خارجية إضافية

### 26.3 إضافة اتصال API جديد

1. انقر على زر **إضافة API**.
2. املأ الحقول المطلوبة:
   - **الاسم** — اسم وصفي لاتصال API
   - **الفئة** — اختر فئة API (Weather، Judicial، Payment Gateway، Other)
   - **عنوان URL الأساسي** — عنوان URL الجذري لـ API الخارجي
   - **مفتاح API** — مفتاح المصادقة للخدمة الخارجية (إن لزم)
   - **الوصف** — وصف مختصر لغرض API
3. انقر على **حفظ** لإنشاء اتصال API.

### 26.4 تعديل اتصال API

1. حدد اتصال API في القائمة.
2. انقر على زر **تعديل** لتعديل تهيئته.
3. حدّث الحقول حسب الحاجة (الاسم، URL، مفتاح API، الفئة، الوصف).
4. انقر على **حفظ** لتطبيق التغييرات.

### 26.5 اختبار اتصالات API

1. حدد اتصال API الذي تريد اختباره.
2. انقر على زر **اختبار الاتصال**.
3. سيحاول النظام الوصول إلى عنوان URL المهيأ والتحقق من الاتصال.
4. ستُعرض رسالة نجاح أو فشل تشير إلى ما إذا كان API قابلاً للوصول.

### 26.6 إدارة فئات API

تساعد فئات API في تنظيم الاتصالات حسب الغرض الوظيفي. عند إضافة أو تعديل اتصال API، اختر الفئة المناسبة للحفاظ على تنظيم التهيئات وسهولة اكتشافها من قبل المسؤولين الآخرين.

## 27. سياسات الاحتفاظ بالبيانات

تتيح وحدة سياسات الاحتفاظ بالبيانات للمسؤولين والمنظمين تحديد مدة الاحتفاظ ببيانات الائتمان لكل بلد، وفقاً للوائح الولاية القضائية.

### 27.1 عرض سياسات الاحتفاظ

1. انتقل إلى **سياسات الاحتفاظ** من قسم الإدارة في الشريط الجانبي (يتطلب وصول Admin أو Regulator).
2. تعرض الصفحة جدولاً بجميع سياسات الاحتفاظ المهيأة، بما في ذلك:
   - **البلد** — الولاية القضائية التي تنطبق عليها السياسة
   - **فترة الأرشفة (أشهر)** — المدة التي تُحفظ فيها البيانات في حالة مؤرشفة قبل اتخاذ إجراء إضافي
   - **فترة المحو (أشهر)** — المدة بعد الأرشفة قبل حذف البيانات نهائياً
   - **الحالة** — ما إذا كانت السياسة نشطة أو غير نشطة
   - **تواريخ الإنشاء/التحديث**

### 27.2 فهم فترات الأرشفة مقابل المحو

- **فترة الأرشفة**: بعد هذا العدد من الأشهر، يتم نقل سجلات الائتمان للبلد المحدد إلى حالة مؤرشفة. لا تُدرج السجلات المؤرشفة في عمليات البحث الائتماني النشطة ولكن لا يزال بالإمكان استرجاعها لأغراض تنظيمية أو تدقيقية.
- **فترة المحو**: بعد هذا العدد الإضافي من الأشهر بعد الأرشفة، يتم حذف البيانات نهائياً (محوها) من النظام. بمجرد المحو، لا يمكن استرداد البيانات.

على سبيل المثال، إذا كان لبلد فترة أرشفة 60 شهراً وفترة محو 24 شهراً، سيتم أرشفة السجلات بعد 5 سنوات وحذفها نهائياً بعد سنتين من الأرشفة (7 سنوات إجمالاً).

### 27.3 إضافة سياسة احتفاظ

1. انقر على زر **إضافة سياسة** في أعلى صفحة سياسات الاحتفاظ.
2. املأ الحقول المطلوبة:
   - **البلد** — أدخل اسم البلد أو الرمز
   - **فترة الأرشفة (أشهر)** — عدد الأشهر قبل أرشفة البيانات
   - **فترة المحو (أشهر)** — عدد الأشهر بعد الأرشفة قبل محو البيانات
3. انقر على **حفظ** لإنشاء سياسة الاحتفاظ.

### 27.4 تعديل سياسة احتفاظ

1. حدد سياسة الاحتفاظ في الجدول.
2. انقر على زر **تعديل** في الصف المقابل.
3. عدّل فترة الأرشفة أو فترة المحو أو إعدادات أخرى حسب الحاجة.
4. انقر على **حفظ** لتطبيق التغييرات.

### 27.5 تشغيل تنفيذ الاحتفاظ

يؤدي زر **تشغيل التنفيذ** إلى بدء عملية تنفيذ الاحتفاظ:

1. انقر على زر **تشغيل التنفيذ** في أعلى صفحة سياسات الاحتفاظ.
2. سيقيّم النظام جميع سياسات الاحتفاظ النشطة مقابل البيانات الحالية.
3. السجلات التي تجاوزت فترة الأرشفة سيتم نقلها إلى حالة مؤرشفة.
4. السجلات التي تجاوزت فترة المحو سيتم حذفها نهائياً.
5. ستعرض رسالة تأكيد نتائج عملية التنفيذ، بما في ذلك عدد السجلات المؤرشفة والمحوة.

**مهم:** تشغيل التنفيذ هو إجراء غير قابل للتراجع بالنسبة للسجلات المحوة. تأكد من تهيئة فترات الاحتفاظ بشكل صحيح قبل تشغيل التنفيذ.

## 28. البحث الشامل

توفر ميزة البحث الشامل تجربة بحث موحدة عبر جميع أنواع الكيانات في النظام.

### 28.1 الوصول إلى البحث الشامل

1. انتقل إلى **البحث الائتماني** من الشريط الجانبي، أو انقر على أيقونة البحث في العنوان.
2. تعرض صفحة البحث شريط بحث مركزي مع قائمة منسدلة لتصفية البلد.

### 28.2 إجراء بحث

1. أدخل مصطلح البحث في حقل البحث (اسم المقترض، رقم الهوية الوطنية، اسم المؤسسة، أو رقم الحساب).
2. اختيارياً اختر **بلداً** من القائمة المنسدلة لتضييق النتائج إلى ولاية قضائية محددة.
3. انقر على **بحث** لتنفيذ الاستعلام.
4. يبحث النظام في وقت واحد عبر:
   - **المقترضون** — مطابقات على الاسم، رقم الهوية الوطنية، رقم جواز السفر، اسم الشركة
   - **المؤسسات** — مطابقات على اسم المؤسسة، رقم التسجيل، البريد الإلكتروني للتواصل
   - **الحسابات الائتمانية** — مطابقات على رقم الحساب، المؤسسة المُقرضة

### 28.3 فهم نتائج البحث

تُعرض النتائج في أقسام مصنفة:

- **المقترضون** — تعرض كل نتيجة صورة الملف الشخصي للمقترض (صورة رمزية مُنشأة تلقائياً أو صورة محمّلة)، الاسم، رقم الهوية الوطنية، النوع (فرد/شركة)، البلد، المدينة، والقطاع. انقر على نتيجة للانتقال إلى صفحة تفاصيل المقترض. يوفر زر "عرض التقرير" وصولاً سريعاً إلى التقرير الائتماني للمقترض.
- **المؤسسات** — تعرض كل نتيجة اسم المؤسسة، رقم التسجيل، البريد الإلكتروني للتواصل، النوع، البلد، والحالة. انقر على نتيجة للانتقال إلى صفحة المؤسسات.
- **الحسابات الائتمانية** — تعرض كل نتيجة المؤسسة المُقرضة، رقم الحساب، نوع الحساب، العملة والرصيد، وحالة الحساب. انقر على نتيجة للانتقال إلى صفحة تفاصيل المقترض المرتبط.

### 28.4 تصفية البلد

تسرد القائمة المنسدلة لتصفية البلد جميع الدول الأفريقية الـ 54 المدعومة. عند اختيار بلد:
- يتم إرجاع النتائج من ذلك البلد فقط.
- يظهر زر "مسح" لإزالة تصفية البلد.
- يعمل المرشح بالتزامن مع مصطلح البحث النصي.

## 29. صور الهوية وتحميل المستندات

يدعم النظام صور الملف الشخصي وتحميل مستندات التعريف للمقترضين.

### 29.1 صور الملف الشخصي المُنشأة تلقائياً

يحصل كل مقترض في النظام تلقائياً على صورة ملف شخصي مُنشأة بواسطة DiceBear. هذه الصور الرمزية:
- فريدة لكل مقترض (مُنشأة من معرّف المقترض كبذرة)
- تُعرض على بطاقات المقترضين في نتائج البحث والقوائم
- تُعرض بشكل بارز في صفحة تفاصيل المقترض
- أنماط مختلفة للمقترضين الأفراد (صور أشخاص) والمقترضين الشركات (أحرف أولى)

### 29.2 تحميل صورة مخصصة

لاستبدال الصورة الرمزية المُنشأة تلقائياً بصورة حقيقية:

1. انتقل إلى صفحة تفاصيل المقترض.
2. مرر المؤشر فوق صورة الملف الشخصي للمقترض في عنوان الصفحة.
3. يظهر **تراكب أيقونة كاميرا** فوق الصورة.
4. انقر على أيقونة الكاميرا لفتح منتقي الملفات.
5. اختر ملف صورة (JPEG أو PNG أو تنسيقات صور قياسية أخرى).
6. **حد حجم الملف:** بحد أقصى **5 ميجابايت** لكل صورة.
7. يتم تحميل الصورة وتستبدل فوراً الصورة الرمزية المُنشأة تلقائياً.
8. يؤكد إشعار نجاح التحميل.

### 29.3 تحميل مستند هوية

لتحميل نسخة ممسوحة ضوئياً من جواز سفر أو هوية وطنية للمقترض:

1. انتقل إلى صفحة تفاصيل المقترض.
2. انقر على زر **تحميل الهوية** في منطقة العنوان أعلى اليمين، **أو** انقر على رسالة التحميل في قسم مستند الهوية في بطاقة المعلومات الشخصية.
3. اختر ملفاً:
   - **التنسيقات المدعومة:** صور (JPEG، PNG) ومستندات PDF
   - **حد حجم الملف:** بحد أقصى **10 ميجابايت** لكل مستند
4. يتم تحميل المستند وعرضه في قسم مستند الهوية في بطاقة المعلومات الشخصية.
5. يؤكد إشعار نجاح التحميل.

### 29.4 عرض واستبدال مستند الهوية

بمجرد تحميل مستند هوية:
- يُعرض كمعاينة صورة في بطاقة المعلومات الشخصية.
- مرر المؤشر فوق معاينة المستند للكشف عن زر **استبدال**.
- انقر على **استبدال** لتحميل مستند جديد، والذي يحل محل المستند السابق.

### 29.5 تخزين الملفات

يتم تخزين الملفات المحمّلة على جانب الخادم:
- تُحفظ الصور في مجلد `uploads/photos/`.
- تُحفظ مستندات الهوية في مجلد `uploads/documents/`.
- تُقدم الملفات عبر مسارات مصادقة — فقط المستخدمون المسجلون يمكنهم الوصول إلى الملفات المحمّلة.
- أسماء الملفات عشوائية للأمان.

## 30. بيئة العرض التوضيحي

توفر بيئة العرض التوضيحي وضع عرض تفاعلي لأصحاب المصلحة والمستثمرين والمقيّمين لاستكشاف النظام دون الحاجة إلى بيانات اعتماد مسبقة التهيئة.

### 30.1 الوصول إلى العرض التوضيحي

1. في صفحة تسجيل الدخول، حدد زر **"جرب العرض التوضيحي التفاعلي"** أسفل نموذج تسجيل الدخول.
2. انقر على الزر للدخول إلى شاشة اختيار العرض التوضيحي.

### 30.2 اختيار دور العرض التوضيحي

تعرض شاشة العرض التوضيحي **3 بطاقات أدوار**، كل منها يمثل منظور مستخدم مختلف:

| الدور | اسم المستخدم | الوصف |
|-------|-------------|-------|
| **Admin** | admin | وصول كامل للنظام — إدارة المستخدمين، الإعدادات، جميع الوحدات |
| **Regulator** | regulator1 | رقابة تنظيمية — لوحة الامتثال، الموافقات، سجلات التدقيق |
| **Bank Officer** | cbe_user | مستخدم مؤسسي — تقديم البيانات الائتمانية، البحث عن المقترضين، تقديم النزاعات |

انقر على أي بطاقة دور لتسجيل الدخول فوراً كذلك المستخدم.

### 30.3 مؤشرات بيئة العرض التوضيحي

عند استخدام بيئة العرض التوضيحي:
- يُعرض **إشعار إخلاء مسؤولية** بخلفية كهرمانية/تحذيرية في صفحة اختيار العرض التوضيحي، لإعلام المستخدمين بأن هذه بيئة عرض توضيحي ببيانات وهمية.
- جميع البيانات في بيئة العرض التوضيحي هي بيانات اختبار مبدئية (102,462 مقترض، 172,359 حساب ائتماني، 3,218 نزاع، 2,147 حكم قضائي عبر 54 دولة أفريقية).

### 30.4 العودة إلى تسجيل الدخول العادي

من شاشة اختيار العرض التوضيحي، انقر على زر **"العودة إلى تسجيل الدخول"** للعودة إلى نموذج تسجيل الدخول القياسي باسم المستخدم/كلمة المرور.

### 30.5 الجولة الإرشادية التفاعلية

بعد تسجيل الدخول عبر بيئة العرض التوضيحي، تنطلق تلقائياً **جولة إرشادية تفاعلية من 11 خطوة** لتعريفك بالميزات الرئيسية للنظام.

**سلوك الجولة:**
1. عند تسجيل الدخول التجريبي، تبدأ الجولة تلقائياً مع تسليط الضوء على عناصر واجهة المستخدم الرئيسية واحداً تلو الآخر.
2. **طبقة تسليط ضوئي** تُعتم بقية الشاشة وتلفت الانتباه إلى العنصر المُبرز حالياً.
3. كل خطوة تعرض تلميحاً وصفياً يشرح الغرض والوظيفة للعنصر المُبرز.

**تغطي خطوات الجولة:**
- التنقل في الشريط الجانبي وتنظيم الوحدات
- بطاقات إحصائيات لوحة المعلومات ووظيفة التفصيل
- رسم نمو المحفظة وتصورات البيانات
- خريطة أفريقيا والتغطية الجغرافية
- وظيفة البحث الائتماني
- إعدادات النظام وخيارات التكوين

**عناصر التحكم في الجولة:**
- **Next** — الانتقال إلى الخطوة التالية
- **Back** — العودة إلى الخطوة السابقة
- **Skip** — إنهاء الجولة فوراً
- **Close** — إغلاق طبقة الجولة الحالية

**إعادة تشغيل الجولة:**
- بعد إكمال الجولة أو تخطيها، يمكنك إعادة تشغيلها في أي وقت بالنقر على زر **"Take a Tour"** المعروض في شعار العرض التوضيحي الكهرماني أعلى الصفحة.

**دعم اللغات:**
- محتوى الجولة متاح بجميع **لغات الاتحاد الأفريقي الخمس** (الإنجليزية، الفرنسية، البرتغالية، العربية، السواحلية).
- تتبع لغة الجولة إعداد لغة الواجهة الحالية.

## الملحق د: مسرد المصطلحات

| المصطلح | التعريف |
|---------|---------|
| **Borrower** | فرد أو كيان شركة حصل على ائتمان من مؤسسة مالية |
| **CDH** | مركز البيانات المركزي — المستودع المركزي لبيانات الائتمان عبر الولايات القضائية |
| **Consent** | تفويض ممنوح من صاحب البيانات (المقترض) للوصول إلى معلوماته الائتمانية |
| **Credit Account** | قرض أو تسهيل ائتماني أو التزام مالي مرتبط بمقترض |
| **Credit Bureau** | منظمة تجمع وتوفر المعلومات الائتمانية |
| **Credit Inquiry** | بحث/طلب للحصول على المعلومات الائتمانية لمقترض |
| **Credit Report** | ملخص شامل لسجل الائتمان والجدارة الائتمانية للمقترض |
| **Credit Score** | قيمة رقمية (300-850) تمثل الجدارة الائتمانية للمقترض |
| **Delinquent** | حساب لديه دفعات متأخرة |
| **Default** | حساب مصنف كغير عامل بسبب عدم الدفع لفترة طويلة |
| **Dispute** | شكوى رسمية حول دقة المعلومات الائتمانية |
| **Four-Eye Principle** | آلية رقابة تتطلب من طرفين مستقلين التحقق من إجراء (صانع-مدقق) |
| **Grace Period** | فترة بعد صرف القرض لا يُطلب خلالها أي دفعات |
| **ISU** | وحدة خدمة الاستعلامات — وظيفة مكتب المساعدة لاستفسارات المستهلكين |
| **Lien** | مطالبة قانونية على ممتلكات تُستخدم كضمان لدين |
| **Maker-Checker** | سير عمل يتطلب من مستخدم إنشاء/تعديل البيانات ومن آخر الموافقة عليها |
| **MFI** | مؤسسة تمويل أصغر |
| **NPL** | قرض غير عامل — قرض يكون فيه المقترض متعثراً أو قريباً من التعثر |
| **PEP** | شخص مكشوف سياسياً — شخص يشغل أو شغل منصباً عاماً بارزاً |
| **RBAC** | التحكم في الوصول بناءً على الأدوار — نموذج أمني يقيد الوصول بناءً على أدوار المستخدمين |
| **Receipt Number** | معرّف فريد يُصدر عند منح الموافقة |
| **Restructured** | قرض تم تعديل شروطه لتقديم تخفيف للمقترض |
| **SACCO** | منظمة ادخار وائتمان تعاونية |
| **Serial Number** | معرّف فريد يُعيَّن لكل تقرير ائتماني مُنشأ |
| **SLA** | اتفاقية مستوى الخدمة — الإطار الزمني المتفق عليه لحل النزاعات |
| **SRS** | مواصفات متطلبات البرمجيات |
| **TIN** | رقم التعريف الضريبي |
| **UAT** | اختبار قبول المستخدم |
| **Written Off** | حساب تم تصنيفه كغير قابل للتحصيل وإزالته من المحافظ النشطة |
