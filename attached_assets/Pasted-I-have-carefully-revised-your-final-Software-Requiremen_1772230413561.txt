## 1. Introduction

### 1.1 Purpose
This Software Requirements Specification defines the complete set of requirements for a unified Central Data Hub (CDH) and Credit Registry System capable of operating across multiple African jurisdictions. The system synthesizes requirements from:

- **Ghana**: Credit Reporting Act, 2007 (Act 726)
- **Ethiopia**: National Bank of Ethiopia Credit Registry Requirements
- **Liberia**: Central Bank of Liberia CRB Regulations #CBL/SD/003/2010
- **Uganda**: Bank of Uganda Central Data Hub RFP (FSD/SRVC/2025/00017)
- **Consolidated**: Cross-jurisdictional requirements matrix (CONS-FR-xx)

### 1.2 Scope
The system serves as a national/regional repository for credit information, enabling:
- Collection and validation of credit data from regulated and non-regulated institutions.
- Secure dissemination to licensed Credit Reference Bureaus (CRBs).
- Regulatory oversight, supervision, and policy analysis by Central Banks.
- Consumer access, dispute resolution, and data rights management.
- Integration with external systems (national ID, courts, collateral registries, tax authorities).

## 2. General Description

### 2.1 System Context
The system operates as the authoritative source for credit information within a jurisdiction, positioned between data providers (financial institutions, utilities, telecoms, courts) and data consumers (CRBs, central bank supervision, approved researchers).

### 2.2 User Characteristics

| User Type | Description | Access Level |
|-----------|-------------|--------------|
| System Administrators | Central Bank IT staff | Full system access, configuration |
| Supervisory Users | Central Bank examiners | All data, analytics, compliance reports |
| Data Providers | Banks, MFIs, ACPs | Submit data, view validation results |
| CRB Users | Licensed credit bureaus | Retrieve validated data via API |
| Consumers | Individual borrowers | View own data, file disputes |
| Helpdesk Staff | Inquiry service unit | Manage disputes, consumer inquiries |

### 2.3 Design Constraints
- Must operate within Microsoft Azure cloud environment (per Uganda RFP).
- Must support both on-premises and cloud deployment (per Liberia).
- Must comply with jurisdictional data sovereignty laws.
- Must support multiple currencies (USD, LRD, GHS, UGX, ETB).
- Must provide complete source code and documentation to client upon completion.

### 2.4 Assumptions and Dependencies
- National ID infrastructure exists and provides API access.
- Data providers have basic technical capacity for structured data submission.
- Central Bank has legal mandate to enforce compliance.
- CRBs will build value-added services on top of base data.
