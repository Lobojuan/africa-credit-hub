Issue: In client/src/components/dispute-chatbot.tsx and client/src/components/public-chatbot.tsx, the auto-scroll useEffect runs before the browser has finished painting the new text DOM nodes, leaving the user looking at the middle of the chat instead of the newest message at the bottom.
Action: In both files, update the scroll useEffect to use a setTimeout to allow the DOM to paint first:

TypeScript
