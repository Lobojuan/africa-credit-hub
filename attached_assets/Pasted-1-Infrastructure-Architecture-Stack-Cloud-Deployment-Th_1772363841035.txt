1. Infrastructure & Architecture Stack
Cloud/Deployment: The system must support Microsoft Azure-native tools (such as Azure Data Lake Storage and Synapse Analytics)  as well as open-source configurations (Linux, PostgreSQL, Nginx) to avoid vendor lock-in depending on the region.

Orchestration: It requires a fault-tolerant microservices architecture backed by a Kubernetes cluster, deployed using Infrastructure-as-Code (Terraform) and automated CI/CD pipelines.

Performance Constraints: The platform must process a minimum of 500 transactions per second (scaling up to 2,000). Crucially, the web interface must be heavily optimized to remain fully functional on network connections as low as 256 kbps to support remote branch users.

2. Core Application Modules
Dual Ingestion Gateways: You need to build parallel ingestion engines: secure B2B APIs (REST/GraphQL secured by OAuth 2.1) for mature banks, and a simplified web portal with drag-and-drop template uploads (CSV, XBRL, JSON) for smaller credit providers.

Algorithmic Entity Matching: An AI-ready deduplication engine that identifies borrowers using disparate data combinations (National ID, Tax ID, name variations, dates of birth, and biometrics) using configurable fuzzy matching rules.

Dynamic Validation Engine: A configurable rules engine that sanitizes all incoming data against predefined standards before committing it to the database.

Governance (Maker-Checker): System-wide enforcement of the "four-eye" principle, meaning any data update, approval, or system configuration initiated by a "maker" must be approved by a separate "checker" before going live.

Dispute Management & Chatbot: A secure portal for consumers and institutions to lodge data disputes, featuring an automated chatbot to guide users and track workflows.

Integrated Billing: A revenue module that connects to national payment switches and mobile money platforms to facilitate online payments and generate automated receipts.

3. Security & Compliance Requirements
Certifications: The deployed system must be designed to pass strict ISO 27001 information security certification audits.

Cryptography: All sensitive data must be encrypted at rest (AES-256) and in transit (TLS 1.2 or higher).

Access Control: The system must enforce phishing-resistant Multi-Factor Authentication (MFA) and strict Role-Based Access Control (RBAC).

Audit Trails: You must implement immutable, tamper-evident forensic logging that captures all system actions, queries, and role changes, which must be exported to secure storage.

4. Crucial Project Phases to Budget For
Discovery & SRS: Finalizing the comprehensive Software Requirements Specification, data validation rules, and architecture workflows prior to coding.

QA & End-to-End (E2E) Testing: Building automated E2E testing frameworks, executing load tests for high concurrency, and passing automated Static and Dynamic Application Security Testing (SAST/DAST) with zero critical OWASP vulnerabilities.

Legacy Data Migration: Engineering Extract, Transform, Load (ETL) pipelines to sanitize, validate, and securely migrate 160 GB of unstructured historical credit data from the central bank into the new system.
